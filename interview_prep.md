# Full Stack Developer Interview Preparation Guide (Next.js Focus)

এই গাইডটি বিশেষভাবে **Nexora Studio** (AI Website Builder / Template Marketplace) এর মতো প্রজেক্টের প্রেক্ষাপটে তৈরি করা হয়েছে। এখানে ২৫০টি গুরুত্বপূর্ণ প্রশ্ন এবং উত্তর দেওয়া হলো যা ইন্টারভিউতে আপনার কনফিডেন্স বাড়াতে সাহায্য করবে।

## 🚀 Category 1: Next.js Backend (30 Questions)

### Q1: Next.js এ Route Handlers কি?
Answer:
Route Handlers আমাদের App Router ব্যবহার করে নির্দিষ্ট route এর জন্য custom request handlers (GET, POST, etc.) তৈরি করতে দেয়। এগুলো মূলত API endpoints হিসেবে কাজ করে। এটি Web standard request এবং response API ব্যবহার করে, যা আগের API routes থেকে অনেক বেশি efficient।

### Q2: Server Actions এবং API Routes এর মধ্যে মূল পার্থক্য কি?
Answer:
Server Actions মূলত client components থেকে direct server-side functions কল করার সুবিধা দেয়, যেখানে আলাদা কোনো API endpoint দরকার হয় না। এটি forms এবং data mutations এর জন্য বেশি উপযোগী। অন্যদিকে, API Routes বা Route Handlers external integrations বা public API এর জন্য বেশি ব্যবহৃত হয়।

### Q3: Next.js Middleware এর কাজ কি?
Answer:
Middleware আমাদের কোনো request complete হওয়ার আগে logic রান করার সুযোগ দেয়। এটি মূলত authentication, redirects, বা headers modify করার জন্য ব্যবহৃত হয়। যেমন: কোনো ইউজার লগইন না থাকলে তাকে `/login` পেজে redirect করা Middleware দিয়ে সহজে করা যায়।

### Q4: NextAuth/Auth.js এ `session` callback এর গুরুত্ব কি?
Answer:
`session` callback এর মাধ্যমে আমরা client-side এ session object এ অতিরিক্ত ডাটা (যেমন: user ID, role) যুক্ত করতে পারি। Default ভাবে শুধু নাম এবং ইমেইল থাকে। প্রজেক্টে Admin role বা User specific ID পাওয়ার জন্য এটি অপরিহার্য।

### Q5: Stripe Webhook কেন গুরুত্বপূর্ণ?
Answer:
Stripe Webhook ব্যবহার করা হয় payment এর status (success, fail) server কে জানানোর জন্য। ইউজার যখন পেমেন্ট করে, Stripe আমাদের নির্দিষ্ট API route এ একটি asynchronous event পাঠায়। এর মাধ্যমে আমরা আমাদের database এ order status update বা template access দিতে পারি।

### Q6: Next.js Backend এ Zod কেন ব্যবহার করা হয়?
Answer:
Zod একটি schema validation library যা API request body বা form data validate করতে সাহায্য করে। এটি নিশ্চিত করে যে server এ আসা ডাটা সঠিক format এ আছে। ভুল ডাটা আসলে এটি automatically error থ্রো করে, যা security এবং stability বাড়ায়।

### Q7: Server-side এ Environment Variables কীভাবে সুরক্ষিত রাখা হয়?
Answer:
Next.js এ যে সব variables এর শুরুতে `NEXT_PUBLIC_` থাকে না, সেগুলো শুধু server-side এ অ্যাক্সেসযোগ্য থাকে। Database URL বা Stripe Secret Key এর মতো sensitive ডাটা কখনো public রাখা উচিত নয়। এটি browser এ leaked হওয়া থেকে রক্ষা করে।

### Q8: API Route এ Rate Limiting কেন দরকার?
Answer:
Rate Limiting ব্যবহার করা হয় যাতে কোনো নির্দিষ্ট ইউজার বা bot অল্প সময়ে অনেক বেশি request পাঠিয়ে server crash না করে। এটি brute-force attacks and API abuse থেকে প্রজেক্টকে রক্ষা করে। Redis ব্যবহার করে এটি খুব সহজে ইমপ্লিমেন্ট করা যায়।

### Q9: Edge Runtime এবং Node.js Runtime এর পার্থক্য কি?
Answer:
Edge Runtime হালকা এবং fast, যা ইউজারের কাছাকাছি geographical location এ রান করে। এটি dynamic content দ্রুত সার্ভ করতে পারে কিন্তু কিছু Node.js API (যেমন: `fs`) এখানে কাজ করে না। Node.js Runtime সব ধরণের package সাপোর্ট করে কিন্তু এটি কিছুটা স্লো হতে পারে।

### Q10: Next.js এ 'use server' ডিরেক্টিভ এর কাজ কি?
Answer:
`'use server'` ডিরেক্টিভ ব্যবহার করে আমরা কোনো function কে Server Action হিসেবে ডিক্লেয়ার করি। এটি Next.js কে বলে দেয় যে এই কোডটি শুধুমাত্র server এ এক্সিকিউট হবে। এটি client components থেকেও কল করা যায়, যা data mutation সহজ করে।

### Q11: API Route এ CORS error কেন হয় এবং কীভাবে সমাধান করা যায়?
Answer:
CORS error হয় যখন এক ডোমেইন থেকে অন্য ডোমেইনের API কল করার চেষ্টা করা হয় এবং server সেটি এলাউ করে না। এটি সমাধান করতে response header এ `Access-Control-Allow-Origin` সেট করতে হয়। Next.js এ `next.config.js` বা custom headers দিয়ে এটি করা যায়।

### Q12: Next.js এ dynamic API routes কীভাবে তৈরি করা হয়?
Answer:
Dynamic API routes তৈরি করতে folder এর নাম square brackets `[id]` দিয়ে দিতে হয়। যেমন: `api/projects/[id]/route.js`। এই folder structure এর মাধ্যমে আমরা `params` থেকে specific ID সংগ্রহ করে database এ query করতে পারি।

### Q13: Stripe Checkout flow কীভাবে কাজ করে?
Answer:
প্রথমে client থেকে server এ request যায় checkout session তৈরি করার জন্য। Server তখন Stripe API কল করে একটি `sessionUrl` রিটার্ন করে। ইউজারকে ওই URL এ redirect করা হয় পেমেন্টের জন্য। পেমেন্ট শেষে Stripe আবার আমাদের সাকসেস পেজে ফিরিয়ে আনে।

### Q14: API Route এ caching কীভাবে control করা যায়?
Answer:
Next.js এ GET requests default ভাবে cache হতে পারে। এটি বন্ধ করতে বা custom সময় সেট করতে `export const dynamic = 'force-dynamic'` বা `revalidate` property ব্যবহার করা হয়। এটি নিশ্চিত করে যে API সব সময় fresh ডাটা রিটার্ন করছে।

### Q15: Next.js Backend এ error handling এর best practice কি?
Answer:
Best practice হলো API endpoint এ try-catch block ব্যবহার করা এবং meaningful error message সহ সঠিক HTTP status code (যেমন: 400, 401, 500) পাঠানো। ইউজারের কাছে sensitive backend error না পাঠিয়ে friendly মেসেজ পাঠানো উচিত।

### Q16: Next.js API তে JSON ডাটা কীভাবে রিসিভ করা হয়?
Answer:
Route handler এর ভেতরে `request.json()` মেথড ব্যবহার করে ডাটা রিসিভ করা হয়। এটি একটি Promise রিটার্ন করে, তাই `await` ব্যবহার করতে হয়। ডাটা পাওয়ার পর সেটি validate করে database operation শুরু করা যায়।

### Q17: Server Actions এ revalidatePath এর কাজ কি?
Answer:
ইউজার যখন কোনো ডাটা আপডেট করে (যেমন: প্রজেক্টের নাম পরিবর্তন), তখন database আপডেট হলেও cache এ পুরোনো ডাটা থাকতে পারে। `revalidatePath` কল করলে Next.js ওই পেজের cache ক্লিয়ার করে নতুন ডাটা ফেচ করে আনে। এটি ইউজারকে instant আপডেট দেখায়।

### Q18: Next.js এ কুপি (Cookies) কীভাবে হ্যান্ডেল করা হয়?
Answer:
Next.js এ `next/headers` থেকে `cookies()` ফাংশন ব্যবহার করে server-side এ কুকি পড়া বা সেট করা যায়। এটি authentication tokens বা user preferences সেভ করার জন্য ব্যবহৃত হয়। Client-side এ এটি সরাসরি access করা যায় না যদি `httpOnly` ফ্ল্যাগ সেট থাকে।

### Q19: Stripe এ recurring payments (subscriptions) কীভাবে হ্যান্ডেল করা হয়?
Answer:
Subscriptions এর জন্য Stripe এ `priceId` এবং `customerId` প্রয়োজন হয়। ইউজারের পেমেন্ট সাকসেস হলে Stripe একটি Subscription ID জেনারেট করে। আমাদের Webhook লিসেন করতে হয় যাতে মাসের শেষে পেমেন্ট ফেল করলে আমরা ইউজারের অ্যাক্সেস বন্ধ করতে পারি।

### Q20: API security তে JWT এবং Session এর মধ্যে কোনটি ভালো?
Answer:
JWT (JSON Web Token) stateless এবং scalable, যা mobile apps বা multiple servers এর জন্য ভালো। অন্যদিকে Session-based auth stateful এবং database এ স্টোর থাকে, যা তাৎক্ষণিক সেশন ক্যানসেল করার জন্য ভালো। NextAuth এ default ভাবে JWT ব্যবহৃত হয় যা সহজ এবং সিকিউর।

### Q21: Next.js এ streaming response কীভাবে কাজ করে?
Answer:
Streaming আমাদের পুরো content তৈরি হওয়ার আগেই ছোট ছোট অংশ (chunks) browser এ পাঠানোর সুবিধা দেয়। এটি বিশেষ করে AI response (যেমন: ChatGPT স্টাইল টেক্সট) বা বড় ডাটা সেট দেখানোর জন্য উপযোগী। এর ফলে ইউজারের waiting time কমে যায়।

### Q22: Webhook signature verification কেন দরকার?
Answer:
Webhook signature verification নিশ্চিত করে যে আমাদের API তে আসা request টি আসলেই Stripe বা ওই সার্ভিস থেকেই এসেছে। এটি ছাড়া যে কেউ fake request পাঠিয়ে আমাদের database এ পেমেন্ট সাকসেস দেখিয়ে দিতে পারে। এটি একটি বড় security measure।

### Q23: Next.js API তে multi-part form data (files) কীভাবে রিসিভ করা হয়?
Answer:
ফাইল আপলোডের জন্য `request.formData()` মেথড ব্যবহার করতে হয়। এটি থেকে specific file বা field গেট করা যায়। এরপর Cloudinary বা S3 এ ফাইলটি পাঠিয়ে তার URL টি database এ সেভ করা হয়।

### Q24: API routes এ authorization কীভাবে চেক করা হয়?
Answer:
প্রথমে `getServerSession` দিয়ে চেক করতে হয় ইউজার লগইন কিনা। এরপর database থেকে তার role (যেমন: Admin) ভেরিফাই করতে হয়। যদি ইউজার Unauthorized হয়, তবে সাথে সাথে ৪-০-৩ status code সহ এরর রিটার্ন করতে হয়।

### Q25: Redis কেন API response এ caching এর জন্য ব্যবহার করা হয়?
Answer:
Redis একটি memory-based database যা অত্যন্ত দ্রুত ডাটা রিড করতে পারে। বারবার PostgreSQL এ query না করে common ডাটা (যেমন: template list) Redis এ cache করে রাখা হয়। এতে database load কমে এবং API response time অনেক ফাস্ট হয়।

### Q26: Next.js এ API versioning কীভাবে করা উচিত?
Answer:
Next.js এ path-based versioning সবচেয়ে কমন, যেমন: `/api/v1/projects`। এটি আমাদের existing ইউজারদের সার্ভিস ডিসরাপ্ট না করে নতুন ফিচার যোগ করতে সাহায্য করে। বড় প্রজেক্টে এটি মেইনটেন্যান্স সহজ করে।

### Q27: Route Handlers এ headers কীভাবে set করা হয়?
Answer:
`NextResponse` এর সাথে headers অবজেক্ট পাস করে custom headers সেট করা যায়। যেমন: Content-Type বা Cache-Control। এটি ব্রাউজার বা client কে বলে দেয় response টি কীভাবে হ্যান্ডেল করতে হবে।

### Q28: Next.js Backend এ Logging এর জন্য কি ব্যবহার করা যেতে পারে?
Answer:
সহজ প্রজেক্টে `console.log` কাজ করলেও বড় প্রজেক্টে Winston বা Pino এর মতো libraries ব্যবহার করা ভালো। এছাড়া Vercel এ deploy করলে inbuilt logs দেখা যায়। প্রডাকশনে এরর ট্র্যাকিং এর জন্য Sentry ব্যবহার করা সবচেয়ে বুদ্ধিমানের কাজ।

### Q29: Database connections কেন serverless এ লিকেজ হতে পারে?
Answer:
Serverless functions বারবার জেনারেট এবং কিল হয়, যা database connections লিমিট শেষ করে দিতে পারে। এটি এড়াতে Prisma Client কে global variable এ রাখতে হয়। এতে একই connection বারবার ব্যবহৃত হয় এবং connection pool শেষ হয় না।

### Q30: Next.js এ background jobs হ্যান্ডেল করার উপায় কি?
Answer:
Next.js direct background jobs সাপোর্ট করে না কারণ এটি serverless এ রান করে। এর জন্য external services যেমন Inngest, Upstash (Redis), বা BullMQ (যদি VPS হয়) ব্যবহার করা হয়। এটি পেমেন্ট রিমাইন্ডার বা বড় ইমেইল ক্যাম্পেইন রান করতে সাহায্য করে।

## 🏗️ Category 2: Next.js Fundamentals (40 Questions)

### Q31: Next.js এ Client Component এবং Server Component এর মূল পার্থক্য কি?
Answer:
Server Component ডিফল্টভাবে সার্ভারে রেন্ডার হয় এবং ব্রাউজারে কোনো জাভাস্ক্রিপ্ট পাঠায় না, যা পারফরম্যান্স উন্নত করে। Client Component ব্রাউজারে ইন্টারঅ্যাক্টিভিটি (যেমন: onClick, state) যোগ করতে ব্যবহৃত হয়। সবসময় চেষ্টা করা উচিত ডাটা ফেচিং সার্ভার কম্পোনেন্টে করার।

### Q32: Hydration বলতে কি বোঝায়?
Answer:
Hydration হলো সার্ভার থেকে আসা স্ট্যাটিক HTML এর সাথে ব্রাউজারে জাভাস্ক্রিপ্ট যুক্ত করে সেটিকে ইন্টারঅ্যাক্টিভ করার প্রক্রিয়া। এর ফলে ইউজার প্রথমে দ্রুত কন্টেন্ট দেখতে পায় এবং পরে জাভাস্ক্রিপ্ট লোড হলে বাটন বা ফর্ম কাজ করা শুরু করে। এটি এসইও এর জন্য খুব কার্যকর।

### Q33: 'use client' ডিরেক্টিভ কখন ব্যবহার করতে হয়?
Answer:
যখন কোনো কম্পোনেন্টে React hooks (useState, useEffect) বা ব্রাউজার স্পেসিফিক এপিআই (window, localStorage) ব্যবহার করার প্রয়োজন হয়, তখন ওই ফাইলের একদম শুরুতে `'use client'` লিখতে হয়। এটি নেক্সট-জেএস কে বলে দেয় যে এই কম্পোনেন্টটি ক্লায়েন্ট সাইডে এক্সিকিউট হবে।

### Q34: App Router এ Layout.js এর কাজ কি?
Answer:
Layout হলো একটি শেয়ারড ইউআই যা একাধিক পেজের মধ্যে কমন থাকে (যেমন: Navbar বা Footer)। এটি পেজ নেভিগেশনের সময় রি-রেন্ডার হয় না, যা স্টেট ধরে রাখতে এবং পারফরম্যান্স বাড়াতে সাহায্য করে। প্রতিটি ফোল্ডারে নিজস্ব লেআউট থাকতে পারে।

### Q35: Next.js এ SEO অপ্টিমাইজেশনের জন্য Metadata API কীভাবে কাজ করে?
Answer:
নেক্সট-জেএস এর App Router এ আমরা `layout.js` বা `page.js` থেকে `metadata` অবজেক্ট এক্সপোর্ট করতে পারি। এটি স্বয়ংক্রিয়ভাবে HTML এর `<head>` সেকশনে টাইটেল, ডেসক্রিপশন এবং ওজি ইমেজ যুক্ত করে দেয়। ডাইনামিক পেজের জন্য `generateMetadata` ফাংশন ব্যবহার করা হয়।

### Q36: Next.js এ Image Component ব্যবহারের সুবিধা কি?
Answer:
`next/image` স্বয়ংক্রিয়ভাবে ইমেজের সাইজ অপ্টিমাইজ করে এবং lazy loading এর সুবিধা দেয়। এটি আধুনিক ফরম্যাটে (যেমন: WebP) ইমেজ সার্ভ করে এবং Layout Shift (CLS) প্রতিরোধ করে। ফলে পেজ লোডিং স্পিড অনেক বেড়ে যায়।

### Q37: SSG (Static Site Generation) কি?
Answer:
SSG হলো বিল্ড টাইমে পুরো পেজটি HTML আকারে তৈরি করে রাখা। এর ফলে ইউজার যখন রিকোয়েস্ট করে, তখন সার্ভারকে নতুন করে ডাটা ফেচ করতে হয় না, সরাসরি স্ট্যাটিক ফাইল সার্ভ করে। এটি নেক্সট-জেএস এর সবচেয়ে দ্রুততম রেন্ডারিং মেথড।

### Q38: ISR (Incremental Static Regeneration) এর কাজ কি?
Answer:
ISR আমাদের পুরো সাইট নতুন করে বিল্ড না করেই স্ট্যাটিক পেজ আপডেট করার সুযোগ দেয়। আমরা একটি `revalidate` সময় সেট করে দিতে পারি, যার পর নেক্সট-জেএস ব্যাকগ্রাউন্ডে ডাটা আপডেট করে নেয়। এটি নিউজ সাইট বা প্রোডাক্ট লিস্টিং এর জন্য আদর্শ।

### Q39: SSR (Server Side Rendering) কেন ব্যবহার করা হয়?
Answer:
SSR এ প্রতিবার ইউজার রিকোয়েস্ট করলে সার্ভার নতুন করে পেজ তৈরি করে পাঠায়। এটি সেই সব পেজের জন্য দরকার যেখানে ডাটা সবসময় ফ্রেশ থাকতে হয় (যেমন: ইউজারের পার্সোনাল ড্যাশবোর্ড)। এটি এসইও এর জন্য ভালো কিন্তু SSG থেকে কিছুটা স্লো।

### Q40: Suspense এবং loading.tsx এর মধ্যে সম্পর্ক কি?
Answer:
`loading.tsx` হলো একটি স্পেশাল ফাইল যা নেক্সট-জেএস এ স্বয়ংক্রিয়ভাবে React Suspense ব্যবহার করে তৈরি হয়। যখন কোনো পেজের কন্টেন্ট লোড হতে সময় নেয়, তখন এই লোডিং ইউআই টি দেখানো হয়। এটি ইউজার এক্সপেরিয়েন্স অনেক স্মুথ করে।

### Q41: Next.js এ error.tsx কেন ব্যবহার করা হয়?
Answer:
`error.tsx` একটি নির্দিষ্ট রুট বা সেকশনের এরর হ্যান্ডেল করার জন্য ব্যবহৃত হয়। যদি কোনো কারণে ডাটা ফেচিং ফেল করে, তবে পুরো অ্যাপ ক্র্যাশ না করে এই এরর বাউন্ডারিটি একটি ফ্রেন্ডলি মেসেজ দেখায়। এখান থেকে 'Try Again' বাটন দিয়ে পুনরায় চেষ্টা করার অপশন দেওয়া যায়।

### Q42: Link Component কেন <a> ট্যাগ থেকে ভালো?
Answer:
`next/link` ক্লায়েন্ট-সাইড নেভিগেশন নিশ্চিত করে, যার ফলে পুরো পেজ রিফ্রেশ হয় না। এটি ব্যাকগ্রাউন্ডে লিংক করা পেজগুলো প্রি-ফেচ (pre-fetch) করে রাখে। এর ফলে ইউজার লিংকে ক্লিক করার সাথে সাথে পেজ লোড হয়ে যায়।

### Q43: Next.js এ ফন্ট অপ্টিমাইজেশন কীভাবে করা হয়?
Answer:
`next/font` ব্যবহার করে আমরা গুগল ফন্ট বা লোকাল ফন্ট লোড করতে পারি। এটি ফন্ট ফাইলগুলোকে বিল্ড টাইমে ডাউনলোড করে নেয় এবং লেআউট শিফট ছাড়াই সার্ভ করে। এটি এক্সটারনাল রিকোয়েস্ট কমায় এবং পারফরম্যান্স বাড়ায়।

### Q44: useRouter hook এর কাজ কি? (App Router এ)
Answer:
`useRouter` হুক আমাদের প্রোগ্রামাটিক নেভিগেশন করার সুবিধা দেয় (যেমন: বাটন ক্লিকে অন্য পেজে যাওয়া)। এর মাধ্যমে আমরা পেজ পুশ (push), রিপ্লেস (replace) বা রিফ্রেশ (refresh) করতে পারি। মনে রাখতে হবে এটি শুধুমাত্র ক্লায়েন্ট কম্পোনেন্টে কাজ করে।

### Q45: Next.js এ Catch-all Routes কি?
Answer:
যদি আমরা একটি ফোল্ডারের নাম `[...slug]` দেই, তবে সেটি ওই পাথের পরের সব সেগমেন্ট ক্যাচ করবে। যেমন: `/docs/intro` এবং `/docs/intro/setup` দুটোই একই ফাইলে রেন্ডার হবে। এটি ডকুমেন্টেশন বা ব্লগের মাল্টি-লেভেল ক্যাটাগরির জন্য খুব উপযোগী।

### Q46: Dynamic segments এ params কীভাবে পাওয়া যায়?
Answer:
ডাইনামিক রুটে (যেমন: `[id]`) সার্ভার কম্পোনেন্ট বা পেজ ফাংশনটি স্বয়ংক্রিয়ভাবে একটি `params` প্রপস পায়। এর ভেতর থেকে আমরা ওই স্পেসিফিক আইডি বা স্লাগটি নিতে পারি। এটি ব্যবহার করে আমরা ডাটাবেস থেকে সঠিক ডাটা ফেচ করি।

### Q47: Next.js এ 'prefetch' প্রপস এর কাজ কি?
Answer:
`<Link>` কম্পোনেন্টে ডিফল্টভাবে `prefetch` ট্রু থাকে। এর মানে হলো ইউজার যখন লিংকের কাছাকাছি যাবে বা ভিউপোর্টে আসবে, নেক্সট-জেএস ওই পেজের কোড আগেই ডাউনলোড করে রাখবে। এটি নেভিগেশনকে ইনস্ট্যান্ট করে তোলে।

### Q48: Next.js এ environment variables (.env) ব্যবহারের নিয়ম কি?
Answer:
সব সিক্রেট ডাটা `.env.local` ফাইলে রাখতে হয়। যেগুলো ব্রাউজারে দরকার সেগুলো `NEXT_PUBLIC_` দিয়ে শুরু করতে হয়। বাকিগুলো শুধু সার্ভারে অ্যাক্সেসযোগ্য থাকে। এটি প্রজেক্টের সিকিউরিটির জন্য অত্যন্ত গুরুত্বপূর্ণ।

### Q49: Next.js এ Parallel Routes কি?
Answer:
Parallel Routes আমাদের একই লেআউটে একাধিক পেজ একসাথে রেন্ডার করার সুবিধা দেয়। এটি ফোল্ডার স্ট্রাকচারে `@` চিহ্ন দিয়ে শুরু করতে হয় (যেমন: `@analytics`)। এটি ড্যাশবোর্ড বা কমপ্লেক্স ইউআই তৈরির জন্য খুব কার্যকর।

### Q50: Intercepting Routes বলতে কি বোঝায়?
Answer:
এটি এমন একটি ফিচার যা আমাদের বর্তমান পেজের কনটেক্সটেই অন্য একটি পেজ লোড করার সুবিধা দেয় (যেমন: গ্যালারি থেকে ইমেজে ক্লিক করলে একটি মোডাল ওপেন হওয়া)। এর ফলে ইউজার রিফ্রেশ করলে সরাসরি ওই পেজে চলে যায় কিন্তু ক্লিক করলে মোডাল হিসেবে দেখে।

### Q51: Next.js এ middleware.js ফাইলটি কোথায় রাখতে হয়?
Answer:
`middleware.js` ফাইলটি প্রজেক্টের রুট ডিরেক্টরিতে (src ফোল্ডার থাকলে তার ভেতরে) রাখতে হয়। এটি শুধুমাত্র একটি ফাইল হিসেবে থাকে এবং সব ইনকামিং রিকোয়েস্টকে প্রসেস করে। এটি রুট লেভেলে কাজ করে বলে পুরো অ্যাপের সিকিউরিটি কন্ট্রোল করা যায়।

### Q52: Static Metadata এবং Dynamic Metadata এর মধ্যে পার্থক্য কি?
Answer:
স্ট্যাটিক মেটাডাটা হলো ফিক্সড ভ্যালু যা সরাসরি অবজেক্ট আকারে লেখা হয়। ডাইনামিক মেটাডাটা `generateMetadata` ফাংশন দিয়ে তৈরি হয় যা ডাটাবেস থেকে আসা ডাটা (যেমন: ব্লগের টাইটেল) অনুযায়ী মেটা ট্যাগ সেট করে। এসইও এর জন্য ডাইনামিক মেটাডাটা খুব জরুরি।

### Q53: Next.js এ custom 404 page কীভাবে তৈরি করা হয়?
Answer:
নেক্সট-জেএস এ `not-found.js` নামে একটি ফাইল রুট বা যেকোনো ফোল্ডারে রাখলে সেটি ওই সেকশনের জন্য কাস্টম ৪-০-৪ পেজ হিসেবে কাজ করবে। যখন কোনো পেজ খুঁজে পাওয়া যাবে না, তখন এটি স্বয়ংক্রিয়ভাবে রেন্ডার হবে।

### Q54: Next.js এ data fetching এর জন্য fetch API ব্যবহারের সুবিধা কি?
Answer:
নেক্সট-জেএস নেটিভ `fetch` এপিআই কে এক্সটেন্ড করেছে যাতে এটি ক্যাচিং (caching) এবং রিভ্যালিডেশন (revalidation) সাপোর্ট করে। এর ফলে একই ডাটা বারবার সার্ভারে রিকোয়েস্ট না পাঠিয়ে ক্যাশ থেকে দ্রুত লোড করা সম্ভব হয়। এটি পারফরম্যান্স অনেক বাড়িয়ে দেয়।

### Q55: Request Memoization কি?
Answer:
এটি এমন একটি প্রক্রিয়া যেখানে একই রেন্ডার সাইকেলে যদি একাধিক জায়গায় একই ফেচ রিকোয়েস্ট থাকে, তবে নেক্সট-জেএস শুধু একবারই রিকোয়েস্ট পাঠায়। এটি সার্ভার লোড কমায় এবং ডাটা কনসিস্টেন্সি বজায় রাখে। এটি শুধুমাত্র সার্ভার কম্পোনেন্টে কাজ করে।

### Q56: Next.js এ Data Cache কীভাবে কাজ করে?
Answer:
নেক্সট-জেএস সার্ভার সাইডে ডাটা ক্যাশ করে রাখে যাতে পরবর্তী রিকোয়েস্টে ডাটাবেসে কল করতে না হয়। এটি `force-cache` বা `revalidate` দিয়ে কন্ট্রোল করা যায়. এটি নিশ্চিত করে যে ডাটা লোডিং টাইম অনেক কমে যায়।

### Q57: Next.js এ Server-side Redirect এবং Client-side Redirect এর মধ্যে পার্থক্য কি?
Answer:
সার্ভার-সাইড রিডাইরেক্ট `redirect()` ফাংশন দিয়ে করা হয় যা সাথে সাথে ইউজারের রিকোয়েস্ট অন্য পাথে পাঠিয়ে দেয়। ক্লায়েন্ট-সাইড রিডাইরেক্ট `router.push()` দিয়ে করা হয় যা ব্রাউজারে পেজ লোড হওয়ার পর কাজ করে। অথেনটিকেশন চেক করার জন্য সার্ভার-সাইড রিডাইরেক্ট বেশি সিকিউর।

### Q58: Next.js এ 'useSearchParams' হুকের কাজ কি?
Answer:
এই হুকটি আমাদের ইউআরএল এর কোয়েরি প্যারামিটার (যেমন: `?search=nexora`) রিড করতে সাহায্য করে। এটি শুধুমাত্র ক্লায়েন্ট কম্পোনেন্টে কাজ করে। ফিল্টারিং বা সার্চ রেজাল্ট দেখানোর জন্য এটি অত্যন্ত প্রয়োজনীয়।

### Q59: Next.js এ Script Component কেন ব্যবহার করা হয়?
Answer:
`next/script` আমাদের থার্ড-পার্টি জাভাস্ক্রিপ্ট (যেমন: Google Analytics) লোডিং অপ্টিমাইজ করতে সাহায্য করে। এটি নির্ধারণ করতে দেয় যে স্ক্রিপ্টটি পেজ লোডের আগে না পরে লোড হবে। এর ফলে মেইন থ্রেড ব্লক হয় না এবং পেজ দ্রুত লোড হয়।

### Q60: Next.js এ layout nesting কীভাবে কাজ করে?
Answer:
একটি ফোল্ডারের ভেতরে অন্য ফোল্ডার থাকলে এবং দুটিতেই `layout.js` থাকলে, নেক্সট-জেএস সেগুলোকে নেস্টেড আকারে রেন্ডার করে। অর্থাৎ রুট লেআউট তার চাইল্ড লেআউটকে র্যাপ (wrap) করবে। এটি ড্যাশবোর্ডের সাব-সেকশন ম্যানেজ করার জন্য খুব ভালো উপায়।

### Q61: Next.js এ 'public' ফোল্ডারের কাজ কি?
Answer:
`public` ফোল্ডারে রাখা ফাইলগুলো (যেমন: লোগো, ইমেজ, রোবটস ডট টেক্সট) সরাসরি রুট ইউআরএল থেকে এক্সেস করা যায়। এগুলো নেক্সট-জেএস কোনো প্রসেসিং ছাড়াই সার্ভ করে। স্ট্যাটিক এসেট রাখার জন্য এটি ব্যবহার করা হয়।

### Q62: Server Components এ কেন event listeners (onClick) কাজ করে না?
Answer:
Server Components শুধুমাত্র সার্ভারে রেন্ডার হয় এবং ব্রাউজারে কোনো জাভাস্ক্রিপ্ট ইন্টারেক্টিভিটি পাঠায় না। যেহেতু ব্রাউজারে কোনো 'JS execution' থাকে না, তাই ইভেন্ট লিসেনার কাজ করে না। ইন্টারেক্টিভিটির জন্য কম্পোনেন্টটিকে ক্লায়েন্ট কম্পোনেন্ট বানাতে হয়।

### Q63: Next.js এ dynamic export এর কাজ কি?
Answer:
যখন কোনো পেজ বা রুটে আমরা `export const dynamic = 'force-dynamic'` লিখি, তখন ওই পেজটি আর স্ট্যাটিক থাকে না। এটি প্রতিবার রিকোয়েস্টে নতুন করে রেন্ডার হয়। যখন কোনো পেজিতে সবসময় লেটেস্ট ডাটা দরকার হয়, তখন এটি ব্যবহার করা হয়।

### Q64: Next.js এ Robots.txt কীভাবে জেনারেট করা হয়?
Answer:
আমরা রুট ফোল্ডারে `robots.js` নামে একটি ফাইল তৈরি করে প্রোগ্রামাটিকালি রোবটস রুলস সেট করতে পারি। এটি সার্চ ইঞ্জিন বটদের বলে দেয় কোন পেজগুলো ইনডেক্স করতে হবে আর কোনগুলো না। এসইও এর জন্য এটি একটি এডভান্সড ফিচার।

### Q65: Next.js এ Server Component থেকে ডাটা ফেচিং কেন ভালো?
Answer:
সার্ভার কম্পোনেন্টে ডাটা ফেচ করলে ডাটাবেসের কাছাকাছি থাকা যায় এবং ব্রাউজারে কোনো এপিআই কী বা সেনসিটিভ কোড ফাঁস হওয়ার ভয় থাকে না। এছাড়া এটি ইউজারের ডিভাইসে কোনো অতিরিক্ত জাভাস্ক্রিপ্ট লোড দেয় না, যা লোডিং স্পিড বাড়ায়।

### Q66: Next.js এ hydration mismatch error কেন হয়?
Answer:
এই এরর তখন হয় যখন সার্ভার থেকে আসা HTML এবং ক্লায়েন্টে রেন্ডার হওয়া প্রথম HTML আলাদা হয় (যেমন: ডেট বা টাইম স্ট্যাম্প)। এটি এড়াতে `useEffect` এর ভেতরে ডাইনামিক ডাটা রেন্ডার করা উচিত অথবা ইউজারের টাইমজোন চেক করে রেন্ডার করা উচিত।

### Q67: Next.js এ SVG ফাইল কীভাবে ব্যবহার করা ভালো?
Answer:
এসভিজি ফাইলগুলো `<img>` ট্যাগে ব্যবহার করা যায় অথবা সরাসরি কম্পোনেন্ট হিসেবে ইমপোর্ট করা যায়। কম্পোনেন্ট হিসেবে ব্যবহার করলে সিএসএস দিয়ে কালার বা সাইজ কন্ট্রোল করা সহজ হয়। অনেক সময় SVGR লাইব্রেরি ব্যবহার করে এটি অটোমেট করা হয়।

### Q68: Next.js এ build process এ কি কি হয়?
Answer:
বিল্ড প্রসেসে নেক্সট-জেএস কোড অপ্টিমাইজ করে, টাইপ চেক করে, ইমেজ অপ্টিমাইজ করে এবং স্ট্যাটিক পেজগুলো জেনারেট করে। এটি সব জাভাস্ক্রিপ্ট ফাইলকে ছোট (minify) করে ফেলে যাতে দ্রুত লোড হয়। শেষে এটি একটি `.next` ফোল্ডার তৈরি করে যা প্রোডাকশনে সার্ভ হয়।

### Q69: Next.js এ 'usePathname' হুকের কাজ কি?
Answer:
এই হুকটি আমাদের বর্তমান পেজের পাথ (path) বা ইউআরএল সেগমেন্ট দেয় (যেমন: `/dashboard/projects`)। এটি ব্যবহার করে আমরা নেভবারে একটিভ লিংক (active link) হাইলাইট করতে পারি। এটি শুধুমাত্র ক্লায়েন্ট কম্পোনেন্টে কাজ করে।

### Q70: Next.js এ caching ক্লিয়ার করার উপায় কি?
Answer:
ডাটা ক্যাশ ক্লিয়ার করতে `revalidatePath` বা `revalidateTag` ব্যবহার করা হয়। এছাড়া ব্রাউজার লেভেলে হার্ড রিফ্রেশ দিলে রাউটার ক্যাশ ক্লিয়ার হয়। বিল্ড টাইমে ক্যাশ ক্লিয়ার করতে `.next` ফোল্ডার ডিলিট করে নতুন করে বিল্ড দিতে হয়।

## 🗄️ Category 3: PostgreSQL Database (15 Questions)

### Q71: PostgreSQL কেন আধুনিক SaaS অ্যাপ্লিকেশনের জন্য জনপ্রিয়?
Answer:
PostgreSQL একটি অত্যন্ত শক্তিশালী ওপেন-সোর্স রিলেশনাল ডাটাবেস যা ডাটা কনসিস্টেন্সি এবং স্কেলেবিলিটির জন্য পরিচিত। এটি এডভান্সড ফিচার যেমন JSONB সাপোর্ট এবং ACID কমপ্লায়েন্স প্রদান করে। নেক্সোরা স্টুডিওর মতো বড় প্রজেক্টে কমপ্লেক্স কুয়েরি হ্যান্ডেল করার জন্য এটি আদর্শ।

### Q72: ACID Properties বলতে কি বোঝায়?
Answer:
ACID হলো Atomicity, Consistency, Isolation, এবং Durability। এটি নিশ্চিত করে যে ডাটাবেস ট্রানজ্যাকশনগুলো সঠিকভাবে সম্পন্ন হবে। এর ফলে সিস্টেম ক্র্যাশ করলেও ডাটা হারানো বা ভুল ডাটা সেভ হওয়ার সম্ভাবনা থাকে না, যা পেমেন্ট বা ইউজার ডাটার জন্য অত্যন্ত জরুরি।

### Q73: Primary Key এবং Unique Key এর মধ্যে পার্থক্য কি?
Answer:
Primary Key একটি টেবিলের প্রতিটি রো কে আলাদাভাবে চেনার জন্য ব্যবহৃত হয় এবং এটি কখনো নাল (null) হতে পারে না। অন্যদিকে Unique Key নিশ্চিত করে যে ওই কলামে কোনো ডুপ্লিকেট ডাটা নেই, তবে এটি নাল ভ্যালু নিতে পারে। একটি টেবিলে একটিই প্রাইমারি কি থাকে কিন্তু একাধিক ইউনিক কি থাকতে পারে।

### Q74: Database Indexing কেন ব্যবহার করা হয়?
Answer:
Indexing ডাটা রিড করার স্পিড অনেক বাড়িয়ে দেয়। এটি অনেকটা বইয়ের সূচিপত্রের মতো কাজ করে, যার ফলে ডাটাবেসকে পুরো টেবিল সার্চ না করে দ্রুত নির্দিষ্ট ডাটা খুঁজে পেতে সাহায্য করে। তবে অতিরিক্ত ইনডেক্স ব্যবহারের ফলে ডাটা রাইট বা ইনসার্ট করার সময় কিছুটা স্লো হতে পারে।

### Q75: PostgreSQL এ JSONB ডাটা টাইপ এর সুবিধা কি?
Answer:
JSONB আমাদের রিলেশনাল ডাটাবেসের ভেতরেই আন-স্ট্রাকচারড ডাটা (যেমন: ইউজারের কাস্টম সেটিংস) সেভ করার সুবিধা দেয়। এটি সাধারণ JSON টাইপ থেকে ফাস্ট কারণ এটি ডাটা বাইনারি ফরম্যাটে সেভ করে এবং এখানে ইনডেক্সিং করা সম্ভব। এটি ফ্লেক্সিবিলিটি এবং পারফরম্যান্সের এক দারুণ কম্বিনেশন।

### Q76: Inner Join এবং Left Join এর মধ্যে পার্থক্য কি?
Answer:
Inner Join শুধুমাত্র সেই ডাটাগুলো রিটার্ন করে যা উভয় টেবিলেই কমন। Left Join বাম পাশের টেবিলের সব ডাটা এবং ডান পাশের টেবিল থেকে শুধু ম্যাচিং ডাটা রিটার্ন করে। নেক্সোরা স্টুডিওতে সব ইউজারের সাথে তাদের প্রজেক্টের লিস্ট পেতে Left Join ব্যবহার করা হয়।

### Q77: Database Normalization কেন প্রয়োজন?
Answer:
Normalization হলো ডাটাবেস ডিজাইনের এমন একটি পদ্ধতি যা ডাটা ডুপ্লিকেট হওয়া কমায় এবং ডাটা ইন্টিগ্রিটি বাড়ায়। এর মাধ্যমে বড় টেবিলকে ছোট টেবিলে ভাগ করা হয় এবং লজিক্যাল রিলেশন তৈরি করা হয়। এটি ডাটাবেসের মেইনটেন্যান্স সহজ করে এবং স্টোরেজ বাঁচে।

### Q78: Foreign Key এর গুরুত্ব কি?
Answer:
Foreign Key দুটি টেবিলের মধ্যে সম্পর্ক তৈরি করে এবং ডাটার নির্ভুলতা নিশ্চিত করে। যেমন: ইউজারের আইডি ছাড়া কোনো প্রজেক্ট ডাটাবেসে সেভ হতে পারবে না। এটি নিশ্চিত করে যে ডাটা ডিলিট করার সময় রেফারেন্সিয়াল ইন্টিগ্রিটি বজায় থাকে।

### Q79: Connection Pooling বলতে কি বোঝায়? (PgBouncer)
Answer:
Connection Pooling বারবার নতুন ডাটাবেস কানেকশন তৈরি না করে আগে থেকেই কিছু কানেকশন ওপেন করে রাখে এবং সেগুলো পুনরায় ব্যবহার করে। এটি বিশেষ করে সার্ভারলেস এনভায়রনমেন্টে (যেমন: Vercel) অত্যন্ত প্রয়োজনীয় কারণ সেখানে কানেকশন লিমিট দ্রুত শেষ হয়ে যায়।

### Q80: PostgreSQL এ 'Explain Analyze' কেন ব্যবহার করা হয়?
Answer:
এটি একটি কুয়েরি অপ্টিমাইজেশন টুল যা আমাদের দেখায় একটি কুয়েরি রান করতে ডাটাবেস কীভাবে কাজ করছে এবং কতটা সময় নিচ্ছে। এর মাধ্যমে আমরা বুঝতে পারি কোনো ইনডেক্স কাজ করছে কি না বা কুয়েরিতে কোথায় সমস্যা আছে। স্লো কুয়েরি ফিক্স করার জন্য এটি সেরা টুল।

### Q81: Database Transaction কেন ব্যবহার করা হয়?
Answer:
ট্রানজ্যাকশন ব্যবহার করা হয় যখন একাধিক ডাটাবেস অপারেশন একসাথে সম্পন্ন হতে হয়। যদি একটি অপারেশন ফেল করে, তবে সবগুলো অপারেশন রোলব্যাক (rollback) হয়ে যায়। পেমেন্ট প্রসেসিং এর ক্ষেত্রে এটি অত্যন্ত গুরুত্বপূর্ণ যাতে টাকা কাটলেও অর্ডার সেভ না হওয়ার মতো সমস্যা না ঘটে।

### Q82: Group By এবং Having এর কাজ কি?
Answer:
Group By একই ধরণের ডাটাগুলোকে গ্রুপ করার জন্য ব্যবহৃত হয় (যেমন: প্রতি মাসে কতজন ইউজার রেজিস্টার করেছে)। Having ব্যবহৃত হয় ওই গ্রুপের ওপর কন্ডিশন সেট করার জন্য। এটি অ্যাগ্রিগেট ফাংশন (যেমন: SUM, COUNT) এর সাথে কাজ করে।

### Q83: PostgreSQL এ View এর কাজ কি?
Answer:
View হলো একটি ভার্চুয়াল টেবিল যা একটি কমপ্লেক্স কুয়েরির রেজাল্ট সেভ করে রাখে। এটি বারবার বড় কুয়েরি না লিখে সহজেই ওই ভিউ থেকে ডাটা রিড করার সুবিধা দেয়। এটি কোড ক্লিন রাখতে এবং সিকিউরিটি বজায় রাখতে সাহায্য করে।

### Q84: SQL Injection থেকে ডাটাবেসকে কীভাবে রক্ষা করা যায়?
Answer:
SQL Injection থেকে বাঁচতে সরাসরি স্ট্রিং কুয়েরি না লিখে Parameterized Queries বা Prisma এর মতো ORM ব্যবহার করা উচিত। এগুলোতে ডাটাবেস কুয়েরি এবং ইউজারের ইনপুট আলাদাভাবে হ্যান্ডেল করা হয়, ফলে হ্যাকাররা ক্ষতিকর কোড রান করতে পারে না।

### Q85: PostgreSQL এ 'Vacuum' কেন প্রয়োজন?
Answer:
PostgreSQL ডাটা ডিলিট করলেও সরাসরি জায়গা খালি করে না, সেগুলো 'Dead Tuples' হিসেবে থাকে। Vacuum প্রসেসটি এই খালি জায়গাগুলো খুঁজে বের করে এবং পুনরায় ব্যবহারের জন্য রিলিজ করে। এটি ডাটাবেস পারফরম্যান্স ঠিক রাখার জন্য একটি নিয়মিত মেইনটেন্যান্স কাজ।

## 💎 Category 4: Prisma ORM (15 Questions)

### Q86: Prisma ORM ব্যবহারের প্রধান সুবিধা কি?
Answer:
Prisma আমাদের ডাটাবেস অপারেশনগুলোকে টাইপ-সেফ (Type-safe) করে তোলে, যা রানটাইম এরর কমায়। এটি `schema.prisma` ফাইলের মাধ্যমে ডাটাবেস ডিজাইন করা সহজ করে এবং অটোমেটিক কুয়েরি বিল্ডার ও মাইগ্রেশন টুল প্রদান করে। এটি ডেভেলপার এক্সপেরিয়েন্স অনেক উন্নত করে।

### Q87: Prisma Migrate এর কাজ কি?
Answer:
Prisma Migrate আমাদের `schema.prisma` ফাইলের পরিবর্তনগুলোকে ডাটাবেস টেবিলে রূপান্তর করে। এটি মাইগ্রেশন হিস্ট্রি ট্র্যাক করে, ফলে টিমের সবাই একই ডাটাবেস স্ট্রাকচার ব্যবহার করতে পারে। প্রোডাকশনে `migrate deploy` কমান্ড ব্যবহার করে নিরাপদভাবে আপডেট করা হয়।

### Q88: Prisma Client কীভাবে জেনারেট করা হয়?
Answer:
যখন আমরা `npx prisma generate` কমান্ড রান করি, তখন প্রিজমা আমাদের স্কিমা অনুযায়ী একটি কাস্টম জাভাস্ক্রিপ্ট ক্লায়েন্ট তৈরি করে। এটি আমাদের কোডে অটো-কমপ্লিশন এবং টাইপ ইন্টেলিসেন্স প্রদান করে। প্রজেক্টের যেকোনো জায়গা থেকে ডাটাবেস কুয়েরি করার জন্য এটি ব্যবহৃত হয়।

### Q89: Prisma এ 'include' এবং 'select' এর মধ্যে পার্থক্য কি?
Answer:
`include` ব্যবহার করা হয় যখন আমরা মূল ডাটার সাথে তার রিলেটেড ডাটা (যেমন: ইউজারের সাথে তার প্রজেক্টগুলো) পুরোটা নিতে চাই। `select` ব্যবহার করা হয় যখন আমরা শুধু নির্দিষ্ট কিছু কলাম (যেমন: শুধু নাম এবং ইমেইল) ফিল্টার করে আনতে চাই। পারফরম্যান্সের জন্য `select` ব্যবহার করা ভালো।

### Q90: Prisma এ Many-to-Many রিলেশন কীভাবে হ্যান্ডেল করা হয়?
Answer:
প্রিজমা ইমপ্লিসিট (implicit) এবং এক্সপ্লিসিট (explicit) দুইভাবেই মেনি-টু-মেনি রিলেশন সাপোর্ট করে। ইমপ্লিসিট রিলেশনে প্রিজমা ব্যাকগ্রাউন্ডে নিজেই একটি জয়েন টেবিল তৈরি করে নেয়। এটি হ্যান্ডেল করা খুব সহজ এবং কোড অনেক ক্লিন থাকে।

### Q91: Prisma Studio কি?
Answer:
Prisma Studio হলো একটি গ্রাফিক্যাল ইউজার ইন্টারফেস (GUI) যা দিয়ে ব্রাউজার থেকেই ডাটাবেসের ডাটা দেখা, এডিট করা বা ডিলিট করা যায়। এটি কোনো থার্ড-পার্টি ডাটাবেস ক্লায়েন্ট ছাড়াই ডাটাবেসের সাথে ইন্টারেক্ট করার একটি আধুনিক উপায়।

### Q92: Prisma এ nested writes বলতে কি বোঝায়?
Answer:
Nested writes এর মাধ্যমে আমরা একটি সিঙ্গেল কুয়েরিতেই প্যারেন্ট এবং চাইল্ড টেবিলের ডাটা ইনসার্ট করতে পারি। যেমন: ইউজার তৈরির সময় একই সাথে তার প্রথম প্রজেক্টটিও তৈরি করে ফেলা। এটি কোড কমায় এবং ডাটা কনসিস্টেন্সি নিশ্চিত করে।

### Q93: Prisma এ raw queries কখন ব্যবহার করা উচিত?
Answer:
যখন প্রিজমার কুয়েরি বিল্ডার দিয়ে কোনো অত্যন্ত কমপ্লেক্স কুয়েরি করা সম্ভব হয় না বা ডাটাবেস স্পেসিফিক কোনো ফিচারের প্রয়োজন হয়, তখন `prisma.$queryRaw` ব্যবহার করা হয়। তবে এটি যথাসম্ভব এড়িয়ে চলা উচিত কারণ এখানে টাইপ-সেফটি পাওয়া যায় না।

### Q94: Prisma এ connection timeout এরর কীভাবে কমানো যায়?
Answer:
এটি এড়াতে ডাটাবেস ইউআরএল এ `connection_limit` প্যারামিটার সেট করা যেতে পারে। এছাড়া নেক্সট-জেএস এ গ্লোবাল প্রিজমা ইনস্ট্যান্স ব্যবহার করলে কানেকশন লিকেজ কমে। বড় প্রজেক্টে এক্সিলারেট (Prisma Accelerate) বা কানেকশন পুলার ব্যবহার করা সবচেয়ে ভালো সমাধান।

### Q95: Prisma Schema এ 'Enums' এর কাজ কি?
Answer:
Enums আমাদের নির্দিষ্ট কিছু অপশন ফিক্স করে দিতে সাহায্য করে (যেমন: ইউজারের রোল হতে পারে শুধু ADMIN বা USER)। এটি ডাটাবেসে ভুল ডাটা ঢোকা বন্ধ করে এবং কোড লেভেলে সাজেশন পেতে সাহায্য করে। এটি ডাটা ইন্টিগ্রিটি বাড়ানোর একটি দারুণ উপায়।

### Q96: Prisma এ pagination করার উপায় কি?
Answer:
প্রিজমায় `take` (কতগুলো ডাটা আসবে) এবং `skip` (কতগুলো ডাটা বাদ যাবে) ব্যবহার করে সহজে অফসেট প্যাগিনেশন করা যায়। এছাড়া বড় ডাটা সেটের জন্য কারসর-বেসড (cursor-based) প্যাগিনেশন ব্যবহার করা হয় যা অনেক বেশি ফাস্ট এবং স্কেলেবল।

### Q97: Prisma এ 'Upsert' মেথড কেন ব্যবহৃত হয়?
Answer:
Upsert হলো Update এবং Insert এর সমন্বয়। এটি চেক করে যদি ডাটাবেসে রেকর্ডটি থাকে তবে আপডেট করে, আর না থাকলে নতুন করে তৈরি করে। এটি ডুপ্লিকেট ডাটা এড়াতে এবং কোড লজিক সহজ করতে খুব কার্যকর।

### Q98: Prisma এ transactions কীভাবে হ্যান্ডেল করা হয়?
Answer:
প্রিজমায় `prisma.$transaction([])` মেথড ব্যবহার করে একাধিক কুয়েরিকে একটি ট্রানজ্যাকশনের আওতায় আনা যায়। যদি ওই অ্যারের যেকোনো একটি কুয়েরি ফেল করে, তবে সব অপারেশন বাতিল হয়ে যায়। এটি ডাটাবেসের বিশুদ্ধতা বজায় রাখে।

### Q99: npx prisma db push এবং prisma migrate dev এর পার্থক্য কি?
Answer:
`db push` মাইগ্রেশন হিস্ট্রি ছাড়াই সরাসরি ডাটাবেস আপডেট করে, যা প্রোটোটাইপিং বা লোকাল ডেভেলপমেন্টের জন্য ভালো। `migrate dev` একটি মাইগ্রেশন ফাইল তৈরি করে এবং হিস্ট্রি বজায় রাখে, যা প্রফেশনাল প্রজেক্টে টিমওয়ার্কের জন্য অপরিহার্য।

### Q100: Prisma এ model filtering (where) কীভাবে কাজ করে?
Answer:
`where` ক্লজের মাধ্যমে আমরা ডাটা ফিল্টার করতে পারি। প্রিজমা এখানে অনেক অপশন দেয় যেমন `contains`, `startsWith`, `lte` (less than or equal) ইত্যাদি। এর ফলে কমপ্লেক্স সার্চ লজিক খুব সহজেই টাইপ-সেফ উপায়ে লেখা যায়।

### Q101: বর্তমানে React Query বা Context API থাকা সত্ত্বেও কেন Redux ব্যবহার করা হয়?

### Q102: Redux Toolkit (RTK) কেন প্রথাগত Redux থেকে ভালো?
Answer:
RTK রেডক্সের কোডিং অনেক সহজ করে দিয়েছে। এটি 'Boilerplate' কোড কমায় এবং ডিফল্টভাবে সেরা প্র্যাকটিসগুলো ফলো করে (যেমন: Immer, Thunk ইন্টিগ্রেশন)। এর মাধ্যমে স্লাইস (Slice) তৈরি করে খুব দ্রুত স্টেট ম্যানেজমেন্ট লজিক লেখা সম্ভব।

### Q103: Redux Store কি?
Answer:
Store হলো পুরো অ্যাপ্লিকেশনের স্টেটের একটি কেন্দ্রীয় কন্টেইনার। এটি একটি জাভাস্ক্রিপ্ট অবজেক্ট যা সব ডাটা ধরে রাখে। আমাদের পুরো অ্যাপে একটি মাত্র স্টোর থাকে এবং সেখান থেকেই সব কম্পোনেন্ট ডাটা রিড করে।

### Q104: Redux এ Actions এবং Reducers এর কাজ কি?
Answer:
Action হলো একটি মেসেজ বা ইভেন্ট যা বলে দেয় স্টেটে কি পরিবর্তন করতে হবে (যেমন: 'ADD_TO_CART')। Reducer হলো একটি পিওর ফাংশন যা ওই অ্যাকশনটি রিসিভ করে এবং স্টেটের নতুন একটি কপি তৈরি করে রিটার্ন করে। রেডক্স সরাসরি স্টেট পরিবর্তন করে না, বরং নতুন স্টেট জেনারেট করে।

### Q105: Redux এ 'Dispatch' ফাংশনটি কেন ব্যবহৃত হয়?
Answer:
Dispatch হলো একমাত্র উপায় যার মাধ্যমে আমরা স্টোরকে কোনো অ্যাকশন পাঠাতে পারি। যখন কোনো বাটনে ক্লিক করা হয় বা এপিআই থেকে ডাটা আসে, আমরা ওই ডাটাটি ডিসপ্যাচ করি যাতে রিডিউসার সেটি প্রসেস করে স্টেট আপডেট করতে পারে।

### Q106: Redux Toolkit এ Slice বলতে কি বোঝায়?
Answer:
Slice হলো একটি অ্যাপ্লিকেশনের নির্দিষ্ট অংশের স্টেট এবং রিডিউসার লজিকের সমষ্টি। যেমন: ইউজারের জন্য একটি 'authSlice' এবং কার্টের জন্য একটি 'cartSlice'। এটি কোডকে মডুলার রাখতে এবং মেইনটেইন করতে সাহায্য করে।

### Q107: useSelector এবং useDispatch হুকের কাজ কি?
Answer:
`useSelector` ব্যবহার করে আমরা রেডক্স স্টোর থেকে নির্দিষ্ট ডাটা রিড করি। `useDispatch` ব্যবহার করে আমরা অ্যাকশন ডিসপ্যাচ করি। এই হুকগুলো রিয়্যাক্ট কম্পোনেন্টের সাথে রেডক্সের কানেকশন তৈরি করে দেয়।

### Q108: Redux এ Async লজিক কীভাবে হ্যান্ডেল করা হয়? (createAsyncThunk)
Answer:
রেডক্স ডিফল্টভাবে অ্যাসিঙ্ক্রোনাস কাজ করতে পারে না। এর জন্য `createAsyncThunk` ব্যবহার করা হয় যা এপিআই কল করার সুবিধা দেয় এবং রিকোয়েস্টের অবস্থা অনুযায়ী (pending, fulfilled, rejected) অটোমেটিক অ্যাকশন ডিসপ্যাচ করে। এটি ডাটা লোডিং স্টেট ম্যানেজ করা সহজ করে।

### Q109: Immer library কেন Redux Toolkit এ ইন-বিল্ট থাকে?
Answer:
ইমার (Immer) আমাদের রেডক্স স্টেটকে সরাসরি মিউটেট (mutate) করার মতো কোড লিখতে দেয় (যেমন: state.push), কিন্তু ব্যাকগ্রাউন্ডে এটি স্টেটের একটি নতুন কপি তৈরি করে। এটি ডেভেলপারদের জন্য কোড লেখা সহজ করে এবং ইমিউটেবিলিটি বজায় রাখে।

### Q110: Redux DevTools এর গুরুত্ব কি?
Answer:
এটি একটি ব্রাউজার এক্সটেনশন যা দিয়ে অ্যাপের স্টেটের প্রতিটি পরিবর্তন 'Time Travel' এর মাধ্যমে দেখা যায়। কোন অ্যাকশনের কারণে স্টেট কি পরিবর্তন হয়েছে তা ট্রেস করা যায়। এটি কমপ্লেক্স বাগ ফিক্স করার জন্য অত্যন্ত প্রয়োজনীয় একটি টুল।

### Q111: Global State এবং Local State এর মধ্যে পার্থক্য কি?
Answer:
Local State (useState) শুধু একটি কম্পোনেন্ট বা তার চাইল্ডে সীমাবদ্ধ থাকে। Global State (Redux) পুরো অ্যাপের যেকোনো জায়গা থেকে অ্যাক্সেস করা যায়। যখন কোনো ডাটা (যেমন: ইউজারের লগইন স্ট্যাটাস) সব পেজে দরকার হয়, তখন গ্লোবাল স্টেট ব্যবহার করা উচিত।

### Q112: Redux Persist কেন ব্যবহার করা হয়?
Answer:
রেডক্স স্টোর ডিফল্টভাবে পেজ রিফ্রেশ করলে মুছে যায়। Redux Persist স্টেটের ডালাকে লোকাল স্টোরেজে সেভ করে রাখে যাতে রিফ্রেশ দিলেও ডাটা (যেমন: কার্ট আইটেম) হারিয়ে না যায়। এটি ইউজার এক্সপেরিয়েন্স অনেক উন্নত করে।

### Q113: Redux এ Provider এর কাজ কি?
Answer:
Provider হলো একটি হাই-অর্ডার কম্পোনেন্ট যা পুরো রিয়্যাক্ট অ্যাপকে রেডক্স স্টোরের সাথে যুক্ত করে। এটি অ্যাপের একদম রুট লেভেলে থাকে এবং নিশ্চিত করে যে সব কম্পোনেন্ট স্টোর এক্সেস করতে পারবে।

### Q114: Redux এ Middleware কি?
Answer:
Middleware হলো এমন একটি লেয়ার যা অ্যাকশন ডিসপ্যাচ হওয়ার পর এবং রিডিউসারে পৌঁছানোর আগে কাজ করে। এটি লগিং (logging), ক্র্যাশ রিপোর্টিং বা অ্যাসিঙ্ক্রোনাস এপিআই কল করার জন্য ব্যবহৃত হয়। 'Redux Thunk' হলো সবচেয়ে কমন মিডলওয়্যার।

### Q115: Redux Toolkit এ ExtraReducers এর কাজ কি?
Answer:
ExtraReducers ব্যবহার করা হয় যখন কোনো স্লাইস তার বাইরের কোনো অ্যাকশনের (যেমন: থাঙ্ক বা অন্য স্লাইসের অ্যাকশন) ওপর ভিত্তি করে স্টেট পরিবর্তন করতে চায়। এটি বিভিন্ন স্লাইসের মধ্যে কানেকশন তৈরি করতে সাহায্য করে।

### Q116: Redux এ PayloadAction টাইপটি কেন জরুরি? (TypeScript Context)
Answer:
PayloadAction নিশ্চিত করে যে অ্যাকশনের সাথে আসা ডাটা (payload) সঠিক টাইপের। এটি কোড লেখার সময় টাইপ সাজেস্ট করে এবং ভুল ডাটা পাস করলে এরর দেখায়। টাইপস্ক্রিপ্ট ব্যবহার করলে এটি প্রজেক্টের স্ট্যাবিলিটি বাড়ায়।

### Q117: Redux এ State Normalization কি?
Answer:
Normalization হলো ডাটাবেসের মতো রেডক্স স্টেটকেও ফ্ল্যাট স্ট্রাকচারে রাখা। অর্থাৎ নেস্টেড অবজেক্ট না রেখে আইডি দিয়ে ডাটা মেইনটেইন করা। এটি কমপ্লেক্স ডাটা আপডেট করা সহজ করে এবং রি-রেন্ডারিং কমিয়ে পারফরম্যান্স বাড়ায়।

### Q118: Redux এ Memoized Selectors (Reselect) কেন ব্যবহার করা হয়?
Answer:
Memoized selectors নিশ্চিত করে যে স্টোরের ডাটা পরিবর্তন না হলে কম্পোনেন্ট যেন বারবার রি-রেন্ডার না হয়। এটি ক্যালকুলেশন ক্যাশ করে রাখে। বড় অ্যাপ্লিকেশনে এটি পারফরম্যান্স অপ্টিমাইজেশনের একটি অন্যতম পদ্ধতি।

### Q119: Redux এ 'Initial State' এর গুরুত্ব কি?
Answer:
Initial State হলো স্টোরের শুরুর ভ্যালু। এটি অ্যাপ লোড হওয়ার সময় ডিফল্ট ডাটা হিসেবে থাকে। এটি সঠিকভাবে সেট না করলে কম্পোনেন্টে 'undefined' এরর আসতে পারে।

### Q120: Redux vs Context API: কোনটি কখন ব্যবহার করবেন?
Answer:
ছোট প্রজেক্ট বা শুধু থিম/ল্যাঙ্গুয়েজ পরিবর্তনের জন্য Context API যথেষ্ট। কিন্তু বড় প্রজেক্ট যেখানে অনেক ডাটা ফ্লো আছে এবং কমপ্লেক্স লজিক হ্যান্ডেল করতে হয়, সেখানে Redux সেরা। রেডক্সের ডিবাগিং এবং মিডলওয়্যার সাপোর্ট কন্টেক্সট এপিআই থেকে অনেক বেশি শক্তিশালী।

## 🎨 Category 6: HTML & CSS (20 Questions)

### Q121: Semantic HTML কেন এসইও এর জন্য গুরুত্বপূর্ণ?
Answer:
Semantic HTML (যেমন: `<header>`, `<article>`, `<footer>`) সার্চ ইঞ্জিন এবং স্ক্রিন রিডারকে পেজের স্ট্রাকচার বুঝতে সাহায্য করে। এটি কন্টেন্টের অর্থ পরিষ্কার করে, যার ফলে সার্চ ইঞ্জিন সঠিক কিওয়ার্ডের জন্য সাইটকে র‍্যাঙ্ক করতে পারে। এটি ওয়েব অ্যাক্সেসিবিলিটিও উন্নত করে।

### Q122: Flexbox এ `justify-content` এবং `align-items` এর মধ্যে পার্থক্য কি?
Answer:
`justify-content` মেইন এক্সিস (সাধারণত ডানে-বামে) বরাবর আইটেমগুলোকে সাজায়। `align-items` ক্রস এক্সিস (উপরে-নিচে) বরাবর আইটেমগুলোকে এলাইন করে। রেসপন্সিভ লেআউট তৈরিতে এই দুটি প্রপার্টি সবচেয়ে বেশি ব্যবহৃত হয়।

### Q123: CSS Grid এবং Flexbox এর মধ্যে প্রধান পার্থক্য কি?
Answer:
Flexbox মূলত ওয়ান-ডাইমেনショナル (শুধু রো বা শুধু কলাম) লেআউটের জন্য ভালো। CSS Grid টু-ডাইমেনショナル (একই সাথে রো এবং কলাম) কমপ্লেক্স লেআউট তৈরির জন্য ডিজাইন করা হয়েছে। ড্যাশবোর্ড বা কলাম-বেসড গ্রিড সিস্টেমের জন্য গ্রিড সেরা।

### Q124: CSS Specificity বলতে কি বোঝায়?
Answer:
Specificity হলো ব্রাউজারের একটি নিয়ম যা ঠিক করে কোন সিএসএস রুলটি এলিমেন্টের ওপর অ্যাপ্লাই হবে। আইডি (ID) এর প্রায়োরিটি সবচেয়ে বেশি, এরপর ক্লাস (Class), এবং সবশেষে ট্যাগ (Tag)। যদি একাধিক রুল থাকে, তবে যার স্পেসিফিসিটি বেশি সেটিই কার্যকর হয়।

### Q125: Tailwind CSS ব্যবহারের সুবিধা কি?
Answer:
Tailwind আমাদের ইউটিলিটি ক্লাসের মাধ্যমে সরাসরি এইচটিএমএল এই ডিজাইন করার সুবিধা দেয়। এটি সিএসএস ফাইল সাইজ ছোট রাখে (PurgeCSS এর মাধ্যমে) এবং ডিজাইন সিস্টেম মেইনটেইন করা সহজ করে। এটি ডেভেলপমেন্ট স্পিড অনেক বাড়িয়ে দেয়।

### Q126: Mobile-first Design কি?
Answer:
Mobile-first Design হলো প্রথমে ছোট স্ক্রিনের জন্য ডিজাইন করা এবং পরে মিডিয়া কুয়েরি দিয়ে বড় স্ক্রিনের জন্য স্টাইল যোগ করা। এটি পারফরম্যান্স বাড়ায় কারণ মোবাইলে কম সিএসএস লোড হয়। বর্তমান সময়ে গুগল র‍্যাঙ্কিংয়ের জন্য এটি অত্যন্ত জরুরি।

### Q127: CSS এ `rem` এবং `em` ইউনিটের মধ্যে পার্থক্য কি?
Answer:
`rem` রুট এলিমেন্টের (HTML এর ডিফল্ট সাইজ) ওপর ভিত্তি করে কাজ করে, যা পুরো সাইটে কনসিস্টেন্সি বজায় রাখে। `em` তার প্যারেন্ট এলিমেন্টের ফন্ট সাইজের ওপর নির্ভর করে। এক্সেসিবিলিটি এবং স্কেলেবল ডিজাইনের জন্য `rem` ব্যবহার করা বেশি নিরাপদ।

### Q128: Web Accessibility (ARIA) কেন প্রয়োজন?
Answer:
ARIA attributes (যেমন: `aria-label`, `role`) প্রতিবন্ধী ব্যক্তিদের স্ক্রিন রিডার ব্যবহার করে ওয়েবসাইট বুঝতে সাহায্য করে। এটি ওয়েবসাইটকে সবার জন্য ব্যবহারযোগ্য (inclusive) করে তোলে। অনেক দেশে এটি আইনিভাবেও বাধ্যতামূলক।

### Q129: CSS এ Stacking Context (z-index) কীভাবে কাজ করে?
Answer:
Stacking Context ঠিক করে কোন এলিমেন্ট কার ওপরে থাকবে। শুধুমাত্র `z-index` দিলেই কাজ করে না, এলিমেন্টের পজিশন (relative, absolute, etc.) থাকা প্রয়োজন। প্যারেন্ট এলিমেন্টের স্ট্যাকিং কন্টেক্সট চাইল্ড এলিমেন্টের ভিজিবিলিটি লিমিট করে দিতে পারে।

### Q130: CSS Variables (Custom Properties) কেন ব্যবহার করবেন?
Answer:
সিএসএস ভেরিয়েবল আমাদের কালার, ফন্ট বা সাইজ এক জায়গায় ডিফাইন করে পুরো প্রজেক্টে ব্যবহার করার সুযোগ দেয়। এটি থিমিং (যেমন: ডার্ক মোড) ইমপ্লিমেন্ট করা অনেক সহজ করে দেয়। এটি রানটাইমে জাভাস্ক্রিপ্ট দিয়েও পরিবর্তন করা সম্ভব।

### Q131: CSS এ `box-sizing: border-box` এর গুরুত্ব কি?
Answer:
এটি নিশ্চিত করে যে এলিমেন্টের প্যাডিং এবং বর্ডার তার টোটাল উইডথ (width) এর ভেতরেই থাকবে। এটি না থাকলে প্যাডিং দিলে এলিমেন্টের সাইজ বেড়ে গিয়ে লেআউট ভেঙে যেতে পারে। আধুনিক সিএসএস রিসেটে এটি সবসময় ব্যবহার করা হয়।

### Q132: Responsive Design এ Media Queries কীভাবে কাজ করে?
Answer:
Media Queries ব্রাউজারকে বলে দেয় নির্দিষ্ট স্ক্রিন সাইজে (যেমন: ৬৪০ পিক্সেলের নিচে) কোন স্টাইলগুলো অ্যাপ্লাই করতে হবে। এটি কন্টেন্টকে মোবাইল, ট্যাবলেট এবং ডেস্কটপের জন্য মানানসই করে তোলে। এর ফলে ইউজার সব ডিভাইসে ভালো এক্সপেরিয়েন্স পায়।

### Q133: CSS এ Absolute এবং Relative পজিশনের মধ্যে সম্পর্ক কি?
Answer:
`position: relative` এলিমেন্টকে তার স্বাভাবিক জায়গাতেই রাখে কিন্তু চাইল্ড এলিমেন্টের জন্য একটি রেফারেন্স পয়েন্ট তৈরি করে। `position: absolute` তার নিকটতম পজিশনড প্যারেন্টের সাপেক্ষে নিজেকে প্লেস করে। এটি ওভারলে বা কাস্টম আইকন প্লেসমেন্টের জন্য খুব দরকারি।

### Q134: Tailwind CSS এ 'Dark Mode' কীভাবে ইমপ্লিমেন্ট করা হয়?
Answer:
Tailwind এ `dark:` মোডিফায়ার ব্যবহার করে সহজেই ডার্ক মোড স্টাইল যোগ করা যায়। এটি ব্রাউজারের সেটিংস অথবা কোনো সিএসএস ক্লাস (সাধারণত `dark` ক্লাস) এর ওপর ভিত্তি করে কাজ করে। এর মাধ্যমে খুব দ্রুত থিম সুইচিং করা সম্ভব।

### Q135: CSS এ Layout Shift (CLS) কীভাবে কমানো যায়?
Answer:
ইমেজ এবং ভিডিওর জন্য সবসময় উইডথ এবং হাইট আগে থেকে ডিফাইন করে রাখতে হয়। এর ফলে কন্টেন্ট লোড হওয়ার আগে ব্রাউজার ওই জায়গাটি খালি রাখে এবং পেজ হঠাৎ করে নিচে নেমে যায় না। এটি গুগল কোর ওয়েব ভাইটালস (Core Web Vitals) এর জন্য গুরুত্বপূর্ণ।

### Q136: CSS এ Transition এবং Animation এর মধ্যে পার্থক্য কি?
Answer:
Transition হলো দুটি স্টেটের মধ্যে স্মুথ পরিবর্তন (যেমন: হোভারে কালার চেঞ্জ)। Animation হলো আরও কমপ্লেক্স মুভমেন্ট যা `keyframes` দিয়ে কন্ট্রোল করা হয় এবং এটি ইউজারের ইন্টারঅ্যাকশন ছাড়াই চলতে পারে। এনিমেশন দিয়ে লোডিং স্পিনার বা এন্ট্রান্স ইফেক্ট তৈরি করা হয়।

### Q137: CSS Modules কেন ব্যবহার করা হয়?
Answer:
CSS Modules আমাদের সিএসএস ক্লাসগুলোকে লোকাল স্কোপে রাখে, ফলে এক কম্পোনেন্টের স্টাইল অন্য কম্পোনেন্টের সাথে সংঘর্ষ (clash) হয় না। এটি নেক্সট-জেএস এ ডিফল্টভাবে সাপোর্ট করে। এটি বড় প্রজেক্টে সিএসএস মেইনটেইন করা সহজ করে।

### Q138: HTML এ `<head>` ট্যাগের ভেতরে কি কি থাকা উচিত?
Answer:
`<head>` এ মেটা ট্যাগ (charset, viewport), টাইটেল, এসইও মেটাডাটা, ওজি ট্যাগ, এবং এক্সটারনাল স্ক্রিপ্ট বা স্টাইল লিংকেজ থাকা উচিত। এটি ব্রাউজার এবং সার্চ ইঞ্জিনকে পেজ সম্পর্কে প্রাথমিক ধারণা দেয়।

### Q139: CSS এ Viewport Units (vh, vw) এর ব্যবহার কি?
Answer:
`vh` এবং `vw` স্ক্রিনের উচ্চতা এবং প্রস্থের ওপর ভিত্তি করে কাজ করে। যেমন: `100vh` মানে পুরো স্ক্রিনের সমান উচ্চতা। এটি ফুল-স্ক্রিন হিরো সেকশন বা মডেল তৈরি করার জন্য খুব উপযোগী।

### Q140: Tailwind এ কাস্টম কনফিগারেশন (tailwind.config.js) কেন দরকার?
Answer:
যদি প্রজেক্টে কোনো নির্দিষ্ট ব্র্যান্ড কালার, ফন্ট বা স্পেসিং প্রয়োজন হয় যা টেইলউইন্ডের ডিফল্ট লিস্টে নেই, তবে সেটি কনফিগারেশন ফাইলে যোগ করতে হয়। এটি আমাদের নিজস্ব ডিজাইন সিস্টেম তৈরি করতে সাহায্য করে।

## ⚡ Category 7: JavaScript (30 Questions)

### Q141: জাভাস্ক্রিপ্টে 'Closure' কি এবং এর একটি বাস্তব উদাহরণ দিন।
Answer:
Closure হলো এমন একটি ফাংশন যা তার বাইরের স্কোপের ভেরিয়েবলগুলোকে মনে রাখতে পারে, এমনকি বাইরের ফাংশনটি এক্সিকিউট হয়ে যাওয়ার পরেও। যেমন: ডাটা হাইড করা বা private counter তৈরি করার জন্য ক্লোজার ব্যবহার করা হয়। এটি মেমোরিতে ভেরিয়েবলের স্টেট ধরে রাখতে সাহায্য করে।

### Q142: Event Loop কীভাবে কাজ করে?
Answer:
Event Loop হলো জাভাস্ক্রিপ্টের এমন একটি মেকানিজম যা অ্যাসিনক্রোনাস কাজগুলো (যেমন: setTimeout, Fetch) ম্যানেজ করে। এটি মেইন থ্রেড (Call Stack) খালি হওয়া পর্যন্ত অপেক্ষা করে এবং তারপর টাস্ক কিউ (Task Queue) থেকে কাজগুলো কল স্ট্যাকে পাঠায়। এর ফলে সিঙ্গেল-থ্রেডেড হয়েও জাভাস্ক্রিপ্ট মাল্টি-টাস্কিং করতে পারে।

### Q143: `var`, `let`, এবং `const` এর মধ্যে প্রধান পার্থক্য কি?
Answer:
`var` হলো ফাংশন-স্কোপড এবং এটি হোইস্টিং সাপোর্ট করে, যা অনেক সময় বাগ তৈরি করে। `let` এবং `const` হলো ব্লক-স্কোপড। `let` এর ভ্যালু পরিবর্তন করা যায় কিন্তু `const` একবার ডিক্লেয়ার করলে তার রেফারেন্স পরিবর্তন করা যায় না। মডার্ন জেএস এ `var` এড়িয়ে চলা উচিত।

### Q144: Promise কি এবং এর কয়টি স্টেট থাকে?
Answer:
Promise হলো এমন একটি অবজেক্ট যা কোনো অ্যাসিনক্রোনাস অপারেশনের ভবিষ্যৎ রেজাল্ট (সফল বা ব্যর্থ) রিপ্রেজেন্ট করে। এর তিনটি স্টেট থাকে: Pending (কাজ চলছে), Fulfilled (সফল), এবং Rejected (ব্যর্থ)। এটি 'Callback Hell' এড়াতে সাহায্য করে।

### Q145: `this` কিওয়ার্ডটি জাভাস্ক্রিপ্টে কীভাবে কাজ করে?
Answer:
`this` কিওয়ার্ডটি নির্ভর করে ফাংশনটি কীভাবে কল করা হয়েছে তার ওপর। গ্লোবাল স্কোপে এটি উইন্ডো অবজেক্টকে বোঝায়, অবজেক্ট মেথডের ভেতরে এটি ওই অবজেক্টকে বোঝায়। অ্যারো ফাংশনে `this` তার চারপাশের (lexical) স্কোপ থেকে ভ্যালু নেয়।

### Q146: Hoisting বলতে কি বোঝায়?
Answer:
Hoisting হলো জাভাস্ক্রিপ্টের এমন একটি বিহেভিয়ার যেখানে ভেরিয়েবল এবং ফাংশন ডিক্লেয়ারেশনকে কোডের একদম শুরুতে নিয়ে যাওয়া হয়। ফাংশন ডিক্লেয়ারেশন পুরোপুরি হোইস্টেড হয়, কিন্তু `var` দিয়ে করা ভেরিয়েবল শুধু নাম হোইস্ট করে (ভ্যালু undefined থাকে)। `let` এবং `const` ও হোইস্ট হয় কিন্তু 'Temporal Dead Zone' এর কারণে এক্সেস করা যায় না।

### Q147: `map()`, `filter()`, এবং `reduce()` এর মধ্যে পার্থক্য কি?
Answer:
`map` একটি অ্যারের প্রতিটি এলিমেন্ট পরিবর্তন করে নতুন একটি অ্যারে দেয়। `filter` কন্ডিশন ম্যাচ করা এলিমেন্টগুলো নিয়ে নতুন অ্যারে তৈরি করে। `reduce` পুরো অ্যারে প্রসেস করে একটি মাত্র ভ্যালু (যেমন: টোটাল সাম) রিটার্ন করে। এগুলো অরিজিনাল অ্যারেকে পরিবর্তন করে না।

### Q148: Async/Await ব্যবহারের সুবিধা কি?
Answer:
Async/Await হলো প্রমিজ হ্যান্ডেল করার একটি আধুনিক এবং ক্লিন উপায়। এটি এসিনক্রোনাস কোডকে দেখতে সিনক্রোনাস কোডের মতো করে তোলে, যা পড়া এবং ডিবাগ করা সহজ। এটি `try-catch` ব্লকের মাধ্যমে এরর হ্যান্ডেল করা সহজ করে দেয়।

### Q149: Prototypal Inheritance কি?
Answer:
জাভাস্ক্রিপ্টে প্রতিটি অবজেক্টের একটি গোপন লিংক থাকে যাকে প্রোটোটাইপ বলা হয়। যদি কোনো অবজেক্টে কোনো প্রপার্টি না পাওয়া যায়, তবে সেটি তার প্রোটোটাইপ চেইনে সার্চ করে। এর মাধ্যমে এক অবজেক্ট অন্য অবজেক্টের মেথড বা প্রপার্টি ইনহেরিট করতে পারে।

### Q150: Arrow Function এবং Regular Function এর মধ্যে মূল পার্থক্য কি?
Answer:
অ্যারো ফাংশনের নিজস্ব `this` নেই, এটি তার চারপাশের কনটেক্সট থেকে `this` নেয়। এছাড়া অ্যারো ফাংশন `arguments` অবজেক্ট সাপোর্ট করে না এবং এগুলোকে 'constructor' হিসেবে ব্যবহার করা যায় না। ছোট এবং ক্লিন কোড লেখার জন্য অ্যারো ফাংশন জনপ্রিয়।

### Q151: Debouncing এবং Throttling এর মধ্যে পার্থক্য কি?
Answer:
Debouncing হলো কোনো ইভেন্ট বারবার ঘটলে শুধু শেষবার বা নির্দিষ্ট সময় পর একবার ফাংশনটি রান করা (যেমন: সার্চ ইনপুট)। Throttling হলো নির্দিষ্ট সময় অন্তর অন্তর ফাংশনটি রান করা (যেমন: উইন্ডো রিসাইজ বা স্ক্রল)। এগুলো পারফরম্যান্স অপ্টিমাইজেশনে খুব কার্যকর।

### Q152: Deep Clone এবং Shallow Clone এর মধ্যে পার্থক্য কি?
Answer:
Shallow Clone শুধু টপ-লেভেল প্রপার্টি কপি করে, ফলে নেস্টেড অবজেক্টগুলো আগের রেফারেন্সই ধরে রাখে। Deep Clone পুরো অবজেক্টকে ভেতর থেকে কপি করে সম্পূর্ণ নতুন একটি অবজেক্ট তৈরি করে। `JSON.parse(JSON.stringify())` দিয়ে সহজে ডিপ ক্লোন করা যায়।

### Q153: JavaScript এ Memory Leak কীভাবে হতে পারে?
Answer:
অপ্রয়োজনীয় গ্লোবাল ভেরিয়েবল, রিমুভ না করা ইভেন্ট লিসেনার, বা ভুলভাবে ব্যবহার করা ক্লোজার মেমোরি লিক তৈরি করতে পারে। এটি ব্রাউজারের মেমোরি দখল করে রাখে এবং ধীরে ধীরে অ্যাপ্লিকেশন স্লো করে দেয়। নিয়মিত ক্লিন-আপ ফাংশন ব্যবহার করে এটি এড়ানো যায়।

### Q154: `null` এবং `undefined` এর মধ্যে পার্থক্য কি?
Answer:
`undefined` মানে হলো একটি ভেরিয়েবল ডিক্লেয়ার করা হয়েছে কিন্তু কোনো ভ্যালু দেওয়া হয়নি। `null` হলো একটি অ্যাসাইন করা ভ্যালু যা ইচ্ছাকৃতভাবে 'খালি' বোঝাতে ব্যবহৃত হয়। টাইপ চেক করলে `undefined` এর টাইপ `undefined` কিন্তু `null` এর টাইপ আসে `object` (এটি জেএস এর একটি পুরোনো বাগ)।

### Q155: Spread Operator এবং Rest Parameter এর পার্থক্য কি?
Answer:
Spread Operator (`...`) একটি অ্যারে বা অবজেক্টকে ভেঙে আলাদা এলিমেন্ট করে ফেলে (কপি করার জন্য ভালো)। Rest Parameter ফাংশনের অনেকগুলো আর্গুমেন্টকে একটি অ্যারেতে গুছিয়ে নেয়। চিহ্ন একই হলেও তাদের ব্যবহারের জায়গা এবং কাজ আলাদা।

### Q156: Higher Order Function কি?
Answer:
যে ফাংশন অন্য একটি ফাংশনকে আর্গুমেন্ট হিসেবে নেয় অথবা একটি ফাংশন রিটার্ন করে, তাকে Higher Order Function বলে। যেমন: `map`, `filter`, `setTimeout` ইত্যাদি। এটি কোডকে আরও ফ্লেক্সিবিল এবং রি-ইউজেবল করে তোলে।

### Q157: `==` এবং `===` এর মধ্যে কোনটি ব্যবহার করা উচিত এবং কেন?
Answer:
সবসময় `===` (Strict Equality) ব্যবহার করা উচিত কারণ এটি ভ্যালুর সাথে টাইপও চেক করে। `==` টাইপ কনভার্সন (coercion) করার চেষ্টা করে যা অনেক সময় অপ্রত্যাশিত রেজাল্ট দেয়। যেমন: `5 == '5'` ট্রু হলেও `5 === '5'` ফলস।

### Q158: Callback Hell কি এবং কীভাবে এটি এড়ানো যায়?
Answer:
যখন একটি কলব্যাকের ভেতরে আরেকটি কলব্যাক এভাবে অনেকগুলো নেস্টেড হয়ে যায়, তখন তাকে Callback Hell বলে। এটি কোডকে অত্যন্ত জটিল এবং অপাঠ্য করে তোলে। প্রমিজ (Promises) বা Async/Await ব্যবহার করে এটি খুব সহজেই সমাধান করা যায়।

### Q159: JavaScript এ Strict Mode কেন ব্যবহার করা হয়?
Answer:
`'use strict'` লিখলে জাভাস্ক্রিপ্ট ইঞ্জিন আরও কড়াকড়িভাবে কোড চেক করে। এটি অনিচ্ছাকৃত ভুল (যেমন: ডিক্লেয়ার না করে ভেরিয়েবল ব্যবহার) এরর হিসেবে দেখায়। এটি কোডকে আরও সিকিউর এবং অপ্টিমাইজড করতে সাহায্য করে।

### Q160: Destructuring কী এবং এর সুবিধা কি?
Answer:
Destructuring আমাদের অ্যারে বা অবজেক্ট থেকে ডাটা সরাসরি ভেরিয়েবলে নেওয়ার একটি শর্টকাট উপায়। এটি কোডকে ক্লিন রাখে এবং বড় অবজেক্ট থেকে প্রয়োজনীয় ডাটা নেওয়া সহজ করে। যেমন: `const { name } = user;`।

### Q161: Template Literals এর সুবিধা কি?
Answer:
Template Literals (backticks ``) আমাদের স্ট্রিংয়ের ভেতরে সরাসরি ভেরিয়েবল `${variable}` এবং মাল্টি-লাইন স্ট্রিং লেখার সুবিধা দেয়। এটি পুরোনো স্ট্রিং কনক্যাটিনেশন থেকে অনেক বেশি রিডেবল এবং ক্লিন।

### Q162: Module System (ESM vs CommonJS) এর পার্থক্য কি?
Answer:
ESM (`import/export`) হলো আধুনিক স্ট্যান্ডার্ড যা ব্রাউজার এবং নেক্সট-জেএস এ ব্যবহৃত হয়। CommonJS (`require/module.exports`) মূলত পুরোনো Node.js প্রজেক্টে দেখা যায়। ESM এ কোড স্ট্যাটিকালি অ্যানালাইজ করা যায়, ফলে 'Tree Shaking' এর মাধ্যমে ফাইল সাইজ ছোট করা সম্ভব।

### Q163: `fetch` API দিয়ে ডাটা ফেচ করার সময় এরর হ্যান্ডেল করবেন কীভাবে?
Answer:
`fetch` এপিআই শুধুমাত্র নেটওয়ার্ক এরর হলে প্রমিজ রিজেক্ট করে। তাই আমাদের `response.ok` প্রপার্টি চেক করতে হয় দেখার জন্য যে সার্ভার থেকে ৪-০-৪ বা ৫-০-০ এরর এসেছে কি না। সাথে `try-catch` ব্যবহার করা বেস্ট প্র্যাকটিস।

### Q164: JavaScript এ Event Delegation কি?
Answer:
ইভেন্ট ডেলিগেশন হলো প্রতিটি চাইল্ড এলিমেন্টে আলাদা ইভেন্ট লিসেনার না দিয়ে তাদের কমন প্যারেন্টে একটি মাত্র লিসেনার দেওয়া। এটি ইভেন্ট বাবলিং (bubbling) মেকানিজম ব্যবহার করে কাজ করে। এর ফলে মেমোরি বাঁচে এবং ডাইনামিকালি যোগ করা এলিমেন্টগুলোও কাজ করে।

### Q165: JavaScript এ Currying বলতে কি বোঝায়?
Answer:
Currying হলো এমন একটি টেকনিক যেখানে একটি ফাংশন যা একাধিক আর্গুমেন্ট নেয়, তাকে এমনভাবে রূপান্তর করা হয় যাতে সেটি একটি করে আর্গুমেন্ট নেয় এবং নতুন একটি ফাংশন রিটার্ন করে। এটি কোড রি-ইউজেবিলিটি এবং পার্শিয়াল অ্যাপ্লিকেশন তৈরির জন্য দরকারী।

### Q166: `Map` এবং `Object` এর মধ্যে পার্থক্য কি?
Answer:
`Map` যেকোনো ধরণের ডাটা টাইপকে কি (key) হিসেবে নিতে পারে, যেখানে `Object` শুধু স্ট্রিং বা সিম্বল নেয়। `Map` এ ডাটা ইনসার্ট করার অর্ডার বজায় থাকে এবং এটি বড় ডাটা সেটের ক্ষেত্রে অবজেক্ট থেকে কিছুটা ফাস্ট হতে পারে।

### Q167: JavaScript এ Set এর কাজ কি?
Answer:
`Set` হলো ইউনিক ভ্যালুর একটি কালেকশন। এটি কোনো ডুপ্লিকেট ডাটা স্টোর করে না। কোনো অ্যারে থেকে ডুপ্লিকেট রিমুভ করার জন্য `new Set(array)` ব্যবহার করা একটি খুব পপুলার ট্রিক।

### Q168: `setTimeout` এবং `setInterval` এর মধ্যে পার্থক্য কি?
Answer:
`setTimeout` একটি নির্দিষ্ট সময় পর শুধু একবারই ফাংশনটি রান করে। `setInterval` নির্দিষ্ট সময় অন্তর অন্তর বারবার ফাংশনটি রান করতে থাকে। এগুলো ক্লিয়ার করার জন্য `clearTimeout` এবং `clearInterval` ব্যবহার করা হয়।

### Q169: JavaScript এ `instanceof` অপারেটরের কাজ কি?
Answer:
`instanceof` চেক করে কোনো একটি অবজেক্ট নির্দিষ্ট কোনো ক্লাসের বা কন্সট্রাক্টরের কি না। এটি প্রোটোটাইপ চেইন চেক করে ট্রু বা ফলস রিটার্ন করে। টাইপ চেকিং এর ক্ষেত্রে এটি খুব কার্যকর।

### Q170: JavaScript এ Prototypal Chain এর শেষ কোথায়?
Answer:
প্রোটোটাইপ চেইনের শেষ হয় `Object.prototype` এ গিয়ে। এর উপরে গেলে `null` পাওয়া যায়। সব জাভাস্ক্রিপ্ট অবজেক্টই শেষ পর্যন্ত `Object` থেকে ইনহেরিট করে।

### Q171: Next.js App Router কেন Page Router এর চেয়ে ভালো? (Trade-off)
Answer:
App Router এ ডিফল্টভাবে সার্ভার কম্পোনেন্ট ব্যবহার করা হয়, যা ব্রাউজারে জাভাস্ক্রিপ্ট কম পাঠায় এবং পারফরম্যান্স বাড়ায়। এটি নেস্টেড লেআউট এবং স্ট্রিমিং সাপোর্ট করে যা পেজ রাউটারে কঠিন ছিল। তবে এটি শিখতে কিছুটা সময় বেশি লাগে এবং অনেক পুরোনো লাইব্রেরি এখানে সরাসরি কাজ করে না।

### Q172: Prisma ORM ব্যবহার করবেন নাকি Raw SQL? কেন?
Answer:
Prisma টাইপ-সেফটি এবং অটোমেটিক মাইগ্রেশন দেয় যা ডেভেলপমেন্ট স্পিড অনেক বাড়িয়ে দেয় এবং ভুল কুয়েরি হওয়ার ঝুঁকি কমায়। তবে অত্যন্ত কমপ্লেক্স কুয়েরির ক্ষেত্রে Raw SQL বেশি ফ্লেক্সিবিলিটি এবং পারফরম্যান্স দিতে পারে। ছোট বা মাঝারি টিমের জন্য প্রিজমা সেরা চয়েস।

### Q173: Redis Caching কেন প্রজেক্টে যোগ করা উচিত?
Answer:
বারবার ডাটাবেস (PostgreSQL) কুয়েরি করা স্লো এবং কস্টলি হতে পারে। রেডিস মেমোরিতে ডাটা সেভ করে রাখে যা মিলিসেকেন্ডের কম সময়ে রেসপন্স দিতে পারে। এটি ডাটাবেসের ওপর লোড কমায় এবং ইউজারের জন্য অ্যাপ্লিকেশনকে অনেক বেশি ফাস্ট মনে হয়।

### Q174: JWT (JSON Web Token) কেন সেশন-বেসড অথেনটিকেশন থেকে জনপ্রিয়?
Answer:
JWT হলো স্টেটলেস (stateless), অর্থাৎ সার্ভারকে কোনো সেশন ডাটাবেসে সেভ করে রাখতে হয় না। এটি মাল্টিপল সার্ভার বা মাইক্রোসার্ভিসে স্কেল করা সহজ। তবে টোকেন একবার ইস্যু করলে সেটি ক্যানসেল করা কঠিন, যা সেশন-বেসড অথেনটিকেশনে সহজেই করা যায়।

### Q175: SQL (PostgreSQL) এবং NoSQL (MongoDB) এর মধ্যে ট্রেড-অফ কি?
Answer:

সিএসএস ভেরিয়েবল দিয়ে ডার্ক মোড করা অনেক বেশি ক্লিন কারণ আমরা শুধু রুট লেভেলে কালার চেঞ্জ করলেই সব এলিমেন্ট আপডেট হয়ে যায়। টেইলউইন্ড ক্লাস কোডকে কিছুটা বড় করতে পারে। তবে টেইলউইন্ড দিয়ে শুরু করা সহজ।

### Q201: যদি আপনার নেক্সট-জেএস অ্যাপে 'Hydration Mismatch' এরর আসে, তবে কীভাবে ফিক্স করবেন?
Answer:
এই এরর সাধারণত ঘটে যখন সার্ভার এবং ক্লায়েন্টের রেন্ডার করা ডাটা আলাদা হয় (যেমন: টাইম)। এটি ফিক্স করতে আমি চেক করব কোনো ডাইনামিক ডাটা (Date.now()) সরাসরি রেন্ডার হচ্ছে কি না। সমাধান হিসেবে ওই পার্টটুকুকে `useEffect` এর ভেতরে রেন্ডার করব অথবা `suppressHydrationWarning` প্রপস ব্যবহার করব।

### Q202: আপনার প্রজেক্টের একটি এপিআই অনেক স্লো হয়ে গেছে, আপনার ইনভেস্টিগেশন স্টেপগুলো কি হবে?

প্রথমে আমি প্রিজমার কুয়েরিগুলো চেক করব কোনো 'Expensive Query' আছে কি না। `explain analyze` দিয়ে ডাটাবেস পারফরম্যান্স দেখব। এরপর দেখব ডাটা ক্যাশ (Redis) ব্যবহার করা হচ্ছে কি না। সবশেষে দেখব কোনো বড় লুপ বা আন-অপ্টিমাইজড ডাটা প্রসেসিং হচ্ছে কি না।

### Q203: পেমেন্ট সাকসেস হয়েছে কিন্তু ইউজার টেম্পলেট এক্সেস পাচ্ছে না। এটি কীভাবে ট্রাবলশুট করবেন?

### Q204: ইউজার লগইন করার পর পেজ রিফ্রেশ করলে আবার লগআউট হয়ে যাচ্ছে। কারণ কি হতে পারে?
Answer:
এটি সাধারণত সেশন বা কুকি কনফিগারেশন ভুলের কারণে হয়। আমি চেক করব `AUTH_SECRET` সঠিকভাবে সেট করা আছে কি না এবং কুকির `domain` বা `secure` ফ্ল্যাগ প্রোডাকশনে ঠিকমতো কাজ করছে কি না। এছাড়া নেক্সট-অথ এর সেশন ডুরেশন কম থাকলে এমন হতে পারে।

### Q205: ডাটাবেসে 'Too many connections' এরর আসলে সমাধান কি?
Answer:
সার্ভারলেস এনভায়রনমেন্টে প্রিজমা বারবার কানেকশন ওপেন করলে এটি হয়। সমাধান হিসেবে আমি প্রিজমা ক্লায়েন্টকে গ্লোবাল ইনস্ট্যান্স হিসেবে রাখব। এছাড়া `Prisma Accelerate` বা `PgBouncer` ব্যবহার করে কানেকশন পুলিং ইমপ্লিমেন্ট করব যাতে লিমিটেড কানেকশন রি-ইউজ করা যায়।

### Q206: একটি পেজে অনেকগুলো ইমেজ থাকার কারণে লোডিং স্লো হচ্ছে। কীভাবে অপ্টিমাইজ করবেন?
Answer:
আমি অবশ্যই `next/image` কম্পোনেন্ট ব্যবহার করব এবং ইমেজের জন্য নির্দিষ্ট সাইজ এবং 'priority' প্রপস (প্রথম ১-২টি ইমেজের জন্য) সেট করব। এছাড়া ইমেজগুলোকে ক্লাউডিনারি বা সিডিএন থেকে অপ্টিমাইজড ভার্সনে (WebP) লোড করব। লেজি লোডিং (Lazy Loading) এখানে ডিফল্টভাবেই সাহায্য করবে।

### Q207: আপনার ডাটাবেস মাইগ্রেশন প্রোডাকশনে ফেল করেছে। এখন আপনার পদক্ষেপ কি?
Answer:
প্রথমে আমি দেখব ফেল হওয়ার কারণ কি (যেমন: ডুপ্লিকেট ডাটা বা কলাম মিসিং)। এরপর `npx prisma migrate resolve` দিয়ে মাইগ্রেশন স্ট্যাটাস ফিক্স করার চেষ্টা করব অথবা ডাটাবেস ব্যাকআপ থেকে রিস্টোর করে ম্যানুয়ালি ফিক্সড মাইগ্রেশন অ্যাপ্লাই করব। প্রোডাকশনে কাজ করার আগে সবসময় লোকাল ও স্টেজিন এনভায়রনমেন্টে টেস্ট করা উচিত।

### Q208: ইউজার বিল্ডারে কাজ করার সময় হঠাত পেজ রিফ্রেশ হলে ডাটা হারিয়ে যাচ্ছে। কীভাবে এটি সমাধান করবেন?

এই সমস্যা সমাধানের জন্য আমি রেডক্সের পাশাপাশি 'Auto-save' ফিচার ইমপ্লিমেন্ট করব। প্রতি কয়েক সেকেন্ড পর পর ড্রাফট ডাটা ডাটাবেস বা লোকাল স্টোরেজে সেভ হবে। ইউজার যখন আবার আসবে, তখন আমরা ওই ড্রাফট ডাটা রিস্টোর করব।

### Q209: প্রজেক্টে মেমোরি লিক (Memory Leak) হচ্ছে কি না কীভাবে বুঝবেন?
Answer:
আমি ব্রাউজারের 'Performance' এবং 'Memory' ট্যাব ব্যবহার করে দেখব হিপ সাইজ (Heap size) সময়ের সাথে সাথে অস্বাভাবিকভাবে বাড়ছে কি না। যদি কোনো কম্পোনেন্ট আনমাউন্ট হওয়ার পরেও মেমোরি রিলিজ না হয়, তবে বুঝতে হবে `useEffect` এর ক্লিন-আপ ফাংশন বা কোনো গ্লোবাল ভেরিয়েবলে সমস্যা আছে।

### Q210: সার্চ রেজাল্ট টাইপ করার সাথে সাথে এপিআই কল হচ্ছে, যা সার্ভারে লোড বাড়াচ্ছে। সমাধান কি?
Answer:
এর জন্য আমি 'Debouncing' টেকনিক ব্যবহার করব। ইউজার টাইপ করা থামানোর ৩০০-৫০০ মিলিসেকেন্ড পর শুধু একবার এপিআই কল হবে। এতে অপ্রয়োজনীয় এপিআই রিকোয়েস্ট কমবে এবং সার্ভার অনেক ফাস্ট থাকবে।

### Q211: ডাটাবেস থেকে ৫০০০ ডাটা একসাথে রিড করতে গিয়ে অ্যাপ হ্যাং হচ্ছে। আপনি কি করবেন?
Answer:
আমি অবশ্যই 'Server-side Pagination' ইমপ্লিমেন্ট করব। প্রিজমার `take` এবং `skip` ব্যবহার করে প্রতি পেজে ১০-২০টি করে ডাটা আনব। এর ফলে ব্রাউজারের ওপর প্রেসার কমবে এবং ডাটা খুব দ্রুত রেন্ডার হবে।

### Q212: ইউজারের প্রোফাইল পিকচার আপডেট হচ্ছে কিন্তু আগেরটা থেকে যাচ্ছে (Cache issue)। সমাধান কি?
Answer:
এটি সাধারণত ব্রাউজার বা সিডিএন ক্যাশিং এর জন্য হয়। সমাধান হিসেবে আমি ইমেজ ইউআরএল এর শেষে একটি টাইমস্ট্যাম্প বা ইউনিক আইডি (Query String) যুক্ত করে দেব (যেমন: `img.jpg?v=123`)। এতে ব্রাউজার বুঝতে পারবে এটি নতুন ইমেজ এবং রি-লোড করবে।

### Q213: প্রোডাকশনে 'Internal Server Error (500)' দেখাচ্ছে কিন্তু লোকালে সব ঠিক। কীভাবে ডিবাগ করবেন?

আমি প্রোডাকশন লogs (Vercel Logs) চেক করব আসল এরর মেসেজটি দেখার জন্য। বেশিরভাগ ক্ষেত্রে এটি এনভায়রনমেন্ট ভেরিয়েবল মিসিং বা ডাটাবেস কানেকশন এরর এর কারণে হয়। প্রোডাকশন ডাটাবেসের সাথে লোকাল কোড কানেক্ট করেও মাঝে মাঝে ডিবাগ করা যায়।

### Q214: ইউজারের পাসওয়ার্ড রিসেট লিংক ইমেইল যাচ্ছে না। চেক করার স্টেপগুলো কি?

প্রথমে আমি দেখব আমার ইমেইল প্রোভাইডার (Nodemailer/SendGrid) এর কোটা শেষ হয়েছে কি না। এরপর দেখব এসএনটিপি (SMTP) ক্রেডেনশিয়াল সঠিক আছে কি না। সবশেষে চেক করব স্প্যাম ফোল্ডার এবং সার্ভারের ইমেইল সেন্ডিং লজিক।

### Q215: ড্যাশবোর্ড লোড হতে অনেক সময় নিচ্ছে কারণ অনেকগুলো এপিআই কল হচ্ছে। সমাধান কি?
Answer:
আমি এপিআই কলগুলোকে প্যারালাল (Promise.all) করার চেষ্টা করব অথবা ডাটাগুলোকে একটি সিঙ্গেল এপিআই এন্ডপয়েন্টে নিয়ে আসব। এছাড়া ড্যাশবোর্ডের ডাটাগুলো রেডিসে ক্যাশ করে রাখব যাতে প্রতিবার ডাটাবেসে কল না করতে হয়।

### Q216: আপনার অ্যাপে ক্রস-সাইট স্ক্রিপ্টিং (XSS) এর ভয় আছে। কীভাবে আটকাবেন?
Answer:
React ডিফল্টভাবেই ডাটা এস্কেপ (escape) করে যা এক্সএসএস আটকায়। তবে আমি নিশ্চিত করব যে আমি যেন কখনো `dangerouslySetInnerHTML` সরাসরি ইউজারের ইনপুটের সাথে ব্যবহার না করি। এছাড়া ইনপুট ভ্যালিডেশনের জন্য Zod ব্যবহার করব এবং সিকিউরিটি হেডার সেট করব।

### Q217: ইউজার কোনো বড় ফাইল আপলোড করার সময় রিকোয়েস্ট টাইম-আউট হচ্ছে। সমাধান কি?
Answer:
সার্ভারলেস ফাংশনের একটি নির্দিষ্ট টাইম লিমিট থাকে। সমাধান হিসেবে আমি ব্রাউজার থেকে সরাসরি Cloudinary বা S3 এ ফাইল আপলোড করব 'Signed Upload' ব্যবহার করে। এতে ফাইল সার্ভার হয়ে যাবে না এবং টাইম-আউট এর সমস্যা হবে না।

### Q218: একটি কমপ্লেক্স কম্পোনেন্ট রি-রেন্ডার হওয়ার কারণে টাইপিং স্লো হচ্ছে। কি করবেন?
Answer:
আমি `React.memo`, `useMemo` বা `useCallback` ব্যবহার করে অপ্রয়োজনীয় রি-রেন্ডার আটকাব। এছাড়া ওই কম্পোনেন্টের স্টেটগুলোকে আরও ছোট ছোট কম্পোনেন্টে ভাগ করব যাতে শুধু প্রয়োজনীয় অংশটুকুই আপডেট হয়।

### Q219: পেমেন্ট ফেল হওয়ার পর ইউজার পেজ ক্লোজ করে দিয়েছে, এখন তাকে কীভাবে আবার ট্রাই করতে বলবেন?
Answer:
আমি Stripe এর 'Incomplete' পেমেন্ট স্ট্যাটাস ট্র্যাক করব এবং যদি ইউজার ১০-১৫ মিনিটের মধ্যে ফিরে না আসে, তাকে একটি অটোমেটিক ইমেইল রিমাইন্ডার পাঠাব তার চেকআউট সেশন লিংকের সাথে। এটি কনভার্সন রেট বাড়াতে সাহায্য করবে।

### Q220: আপনার প্রজেক্টের সাইজ বড় হওয়ার কারণে বিল্ড টাইম অনেক বেড়ে গেছে। উপায় কি?
Answer:
আমি কোড স্প্লিটিং (Dynamic Imports) ব্যবহার করব যাতে বড় লাইব্রেরিগুলো শুধু প্রয়োজন হলে লোড হয়। এছাড়া নেক্সট-জেএস এর বিল্ড ক্যাশ এবং প্রোডাকশন লেভেলে শুধু প্রয়োজনীয় ডিপেন্ডেন্সি রাখার চেষ্টা করব। অপ্রয়োজনীয় বড় প্যাকেজগুলো চেক করে সেগুলো ছোট অল্টারনেটিভ দিয়ে রিপ্লেস করব।

## 🏆 Category 10: Most Asked Project Interview Questions (30 Questions)

### Q221: আপনার এই প্রজেক্টের (Nexora Studio) আর্কিটেকচার সম্পর্কে সংক্ষেপে বলুন।
Answer:
এটি একটি ফুল-স্ট্যাক নেক্সট-জেএস অ্যাপ্লিকেশন যেখানে অ্যাপ রাউটার (App Router) ব্যবহার করা হয়েছে। ব্যাকএন্ডে পোস্টগ্রেএসকিউএল ডাটাবেস এবং প্রিজমা ওআরএম ব্যবহৃত হয়েছে। অথেনটিকেশনের জন্য নেক্সট-অথ এবং পেমেন্টের জন্য স্ট্রাইপ ইন্টিগ্রেট করা হয়েছে। ফ্রন্টএন্ডে রেডক্স টুলকিট দিয়ে স্টেট ম্যানেজমেন্ট এবং টেইলউইন্ড সিএসএস দিয়ে ডিজাইন করা হয়েছে।

### Q222: এই প্রজেক্টটি তৈরি করতে আপনার সবচেয়ে বড় চ্যালেঞ্জ কি ছিল?
Answer:
সবচেয়ে বড় চ্যালেঞ্জ ছিল একটি ড্রাগ-এন্ড-ড্রপ ওয়েবসাইট বিল্ডার তৈরি করা যা ইউজার ফ্রেন্ডলি হবে এবং একই সাথে ক্লিন কোড জেনারেট করবে। ডাটাবেসে কমপ্লেক্স ওয়েবসাইট লেআউট সেভ করা এবং সেগুলো রিয়াল-টাইমে প্রিভিউ করা বেশ কঠিন ছিল। আমি এটি সমাধান করেছি একটি ডাইনামিক কম্পোনেন্ট রেন্ডারিং সিস্টেম এবং রেডক্স স্টেট সিঙ্ক্রোনাইজেশনের মাধ্যমে।

### Q223: আপনি কেন নেক্সট-জেএস বেছে নিলেন, রিয়্যাক্ট কেন নয়?
Answer:
নেক্সট-জেএস রিয়্যাক্টের ওপর ভিত্তি করে তৈরি হলেও এটি ইন-বিল্ট এসএসআর (SSR) এবং এসএসজি (SSG) সুবিধা দেয়, যা ওয়েবসাইট বিল্ডারের এসইও এবং পারফরম্যান্সের জন্য জরুরি। এর ফাইল-বেসড রাউটিং, এপিআই রুটস এবং ইমেজ অপ্টিমাইজেশন ফিচারগুলো অনেক সময় বাঁচায়। এটি আমাকে একটি সিঙ্গেল কোডবেসেই ফ্রন্টএন্ড এবং ব্যাকএন্ড ম্যানেজ করার সুবিধা দিয়েছে।

### Q224: আপনার প্রজেক্টে অথেনটিকেশন কীভাবে কাজ করে?
Answer:
আমি নেক্সট-অথ (Auth.js) ব্যবহার করেছি যা ইউজারের ক্রেডেনশিয়াল এবং গুগল ও-অথ (OAuth) দুইভাবেই লগইন সাপোর্ট করে। ইউজার যখন লগইন করে, তখন একটি জে-ডব্লিউ-টি (JWT) টোকেন জেনারেট হয় যা সেশন কুকিতে সেভ থাকে। মিডলওয়্যার ব্যবহার করে আমি ড্যাশবোর্ড এবং এডমিন রাউটগুলোকে প্রটেক্ট করেছি।

### Q225: স্ট্রাইপ পেমেন্ট ফ্লো-টি একটু বিস্তারিত ব্যাখ্যা করুন।
Answer:
ইউজার যখন কোনো প্ল্যান বা টেম্পলেট কিনতে চায়, তখন আমার সার্ভার থেকে স্ট্রাইপ চেকআউট সেশন তৈরি করা হয়। ইউজারকে স্ট্রাইপের সিকিউর পেজে পাঠানো হয়। পেমেন্ট শেষ হওয়ার পর স্ট্রাইপ আমার সার্ভারে একটি Webhook পাঠায়। সেই Webhook রিসিভ করে আমি ডাটাবেসে ইউজারের সাবক্রিপশন স্ট্যাটাস আপডেট করি এবং তাকে টেম্পলেট এক্সেস দেই।

### Q226: ডাটাবেস ডিজাইনের ক্ষেত্রে আপনি কি কি মাথায় রেখেছেন?
Answer:
আমি রিলেশনাল ডাটাবেস ডিজাইন ফলো করেছি যেখানে ইউজার, প্রজেক্ট এবং টেম্পলেটের মধ্যে স্পষ্ট সম্পর্ক আছে। ডাটাবেস ইন্টিগ্রিটি বজায় রাখতে আমি ফরেন কি এবং ইনডেক্সিং ব্যবহার করেছি। এছাড়া টেম্পলেটের কমপ্লেক্স লেআউট সেভ করার জন্য আমি পোস্টগ্রেএসকিউএল এর JSONB কলাম ব্যবহার করেছি যা অনেক ফ্লেক্সিবল।

### Q227: প্রজেক্টের পারফরম্যান্স বাড়ানোর জন্য আপনি কি কি পদক্ষেপ নিয়েছেন?
Answer:
পারফরম্যান্সের জন্য আমি নেক্সট-জেএস এর ইমেজ অপ্টিমাইজেশন ব্যবহার করেছি। ডাইনামিক ডাটার জন্য রেডিস ক্যাশিং এবং ডাটাবেস কুয়েরিতে প্যাগিনেশন ইমপ্লিমেন্ট করেছি। এছাড়া ক্লায়েন্ট সাইডে বড় লাইব্রেরিগুলো ডাইনামিক ইমপোর্ট (Lazy Loading) এর মাধ্যমে লোড করেছি যাতে ইনিশিয়াল বান্ডেল সাইজ ছোট থাকে।

### Q228: রেডিস (Redis) আপনার প্রজেক্টে ঠিক কি ভূমিকা পালন করছে?
Answer:
রেডিস মূলত ক্যাশিং এবং রেট লিমিটিং এর জন্য ব্যবহৃত হয়েছে। আমি পপুলার টেম্পলেট লিস্টগুলো রেডিসে সেভ করে রাখি যাতে প্রতিবার ডাটাবেসে হিট না লাগে। এছাড়া এপিআই এরর এবং স্প্যামিং রোধ করতে আমি মিডলওয়্যারে রেডিস-বেসড রেট লিমিটিং ব্যবহার করেছি।

### Q229: প্রজেক্টের সিকিউরিটি নিশ্চিত করতে আপনি কি কি করেছেন?
Answer:
সিকিউরিটির জন্য আমি সব ইনপুট ডাটা Zod দিয়ে ভ্যালিডেট করেছি যাতে এসকিউএল ইনজেকশন বা ক্ষতিকর ডাটা না ঢোকে। অথেনটিকেশনে সিকিউর কুকি এবং সিএসআরএফ (CSRF) প্রটেকশন নিশ্চিত করেছি। এছাড়া সব সেনসিটিভ এনভায়রনমেন্ট ভেরিয়েবল শুধু সার্ভার সাইডে রেখেছি।

### Q230: আপনি প্রজেক্টটি কোথায় এবং কীভাবে ডিপ্লয় করেছেন?
Answer:
আমি ফ্রন্টএন্ড এবং ব্যাকএন্ড Vercel এ ডিপ্লয় করেছি কারণ এটি নেক্সট-জেএস এর জন্য সবচেয়ে ভালো সাপোর্ট দেয়। ডাটাবেস হিসেবে আমি Neon (Postgres) ব্যবহার করেছি যা সার্ভারলেস এবং স্কেলেবল। সিআই/সিডি এর জন্য আমি গিটহাব অ্যাকশন ব্যবহার করি, যার ফলে কোড পুশ করার সাথে সাথে অটোমেটিক ডিপ্লয় হয়ে যায়।

### Q231: আপনি যদি প্রজেক্টটি আজ আবার শুরু করতেন, তবে কি আলাদাভাবে করতেন?
Answer:
আমি শুরু থেকেই TypeScript ব্যবহার করতাম কারণ প্রজেক্টের সাইজ বাড়ার সাথে সাথে জাভাস্ক্রিপ্টে টাইপ রিলেটেড বাগ ম্যানেজ করা কঠিন হয়ে পড়েছিল। এছাড়া আমি আরও বেশি 'Headless UI' কম্পোনেন্ট ব্যবহার করতাম যাতে বিল্ডার সেকশনটি আরও দ্রুত কাস্টমাইজ করা যায়।

### Q232: ইউজারের ওয়েবসাইট পাবলিশিং প্রসেসটি কীভাবে কাজ করে?
Answer:
ইউজার যখন পাবলিশ বাটনে ক্লিক করে, আমি তার ডিজাইন করা JSON লেআউটটিকে একটি ইউনিক স্লাগ (slug) এর সাথে ডাটাবেসে সেভ করি। আমার একটি ডাইনামিক রাউট আছে `/[slug]` যা ওই ডাটা রিড করে রেন্ডার করে। এটি ইউজারের সাইটকে ইনস্ট্যান্টলি পাবলিকলি এভেলেবল করে দেয়।

### Q233: প্রজেক্টে রিডাক্স (Redux) কেন ব্যবহার করেছেন?
Answer:
বিল্ডারের ভেতরে অনেকগুলো কমপ্লেক্স স্টেট ছিল (যেমন: ড্রাগড এলিমেন্ট, কালার প্যালেট, টেক্সট এডিট)। এই ডাটাগুলো অনেকগুলো নেস্টেড কম্পোনেন্টে শেয়ার করতে হচ্ছিল। রিডাক্স আমাকে একটি কেন্দ্রীয় স্টোর দিয়েছে যা এই কমপ্লেক্স ডাটা ফ্লো ম্যানেজ করা এবং ডিবাগ করা সহজ করেছে।

### Q234: আপনি কীভাবে আপনার কোড কোয়ালিটি মেইনটেইন করেন?
Answer:
আমি কোড ক্লিন রাখতে ESLint এবং Prettier ব্যবহার করি। আমি সবসময় অর্থবহ ভেরিয়েবল নাম এবং ছোট ছোট ফাংশন লেখার চেষ্টা করি। এছাড়া প্রতিটি ফিচার শেষ করে আমি ম্যানুয়ালি এবং প্রোডাকশন লogs চেক করে বাগগুলো ফিক্স করি।

### Q235: আপনার প্রজেক্টটি কি এসইও ফ্রেন্ডলি? কীভাবে নিশ্চিত করেছেন?
Answer:
হ্যাঁ, এটি সম্পূর্ণ এসইও ফ্রেন্ডলি। আমি নেক্সট-জেএস এর মেটাডাটা এপিআই ব্যবহার করে প্রতিটি পেজের জন্য ডাইনামিক টাইটেল এবং ডেসক্রিপশন সেট করেছি। এছাড়া ইমেজগুলোতে অল্ট (alt) টেক্সট এবং সাইটম্যাপ জেনারেট করেছি যাতে গুগল সহজে ইনডেক্স করতে পারে।

### Q236: ইমেজ আপলোডের ক্ষেত্রে ক্লাউডিনারি (Cloudinary) ব্যবহারের অভিজ্ঞতা কেমন?
Answer:
Cloudinary আমার কাজ অনেক সহজ করে দিয়েছে। আমি ইউজারের আপলোড করা ইমেজগুলোকে সরাসরি Cloudinary তে পাঠাই এবং সেখান থেকে অপ্টিমাইজড ইউআরএল ডাটাবেসে সেভ করি। এর ফলে আমাকে সার্ভারে ইমেজ স্টোরেজ বা রিসাইজিং নিয়ে চিন্তা করতে হয় না।

### Q237: ড্যাশবোর্ডে ডাটা ভিজ্যুয়ালাইজেশনের জন্য কি ব্যবহার করেছেন?
Answer:
আমি Recharts লাইব্রেরি ব্যবহার করেছি। এটি দিয়ে ইউজারের প্রজেক্টের স্ট্যাটাস, পেমেন্ট হিস্ট্রি এবং এনালিটিক্স খুব সুন্দর গ্রাফ আকারে দেখানো হয়েছে। এটি ড্যাশবোর্ডকে অনেক বেশি প্রফেশনাল লুক দেয়।

### Q238: ইউজার যদি ডুপ্লিকেট অ্যাকাউন্ট খুলতে চায় তবে আপনার সিস্টেম কীভাবে হ্যান্ডেল করে?
Answer:
ডাটাবেস লেভেলে আমি ইমেইল কলামটিকে 'Unique' সেট করেছি। যখন কেউ ডুপ্লিকেট ইমেইল দিয়ে সাইন-আপ করার চেষ্টা করে, প্রিজমা একটি এরর দেয় যা আমি ক্যাচ করে ইউজারকে একটি ফ্রেন্ডলি মেসেজ দেখাই যে এই ইমেইলটি অলরেডি রেজিস্টার্ড।

### Q239: আপনি কি এই প্রজেক্টে কোনো টেস্টিং করেছেন?
Answer:
আমি মূলত ম্যানুয়াল টেস্টিং এবং কিছু ক্রিটিক্যাল এপিআই এন্ডপয়েন্টের জন্য Postman দিয়ে অটোমেটেড টেস্ট করেছি। ভবিষ্যতে আমি Jest এবং Cypress ব্যবহার করে ইউনিট এবং এন্ড-টু-এন্ড টেস্ট যোগ করার পরিকল্পনা করছি।

### Q240: প্রজেক্টের ফিউচার ইমপ্রুভমেন্ট প্ল্যান কি?
Answer:
আমি এখানে এআই-বেসড ওয়েবসাইট জেনারেশন যোগ করতে চাই যেখানে ইউজার শুধু প্রম্পট দিলেই একটি পুরো সাইট তৈরি হবে। এছাড়া মাল্টি-ল্যাংগুয়েজ সাপোর্ট এবং Figma থেকে সরাসরি ইমপোর্ট করার ফিচার যোগ করার ইচ্ছা আছে।

### Q241: আপনার প্রজেক্টে পাসওয়ার্ড কীভাবে স্টোর করা হয়?
Answer:
ইউজারের পাসওয়ার্ড কখনোই প্লেইন টেক্সট হিসেবে সেভ হয় না। আমি `bcryptjs` লাইব্রেরি ব্যবহার করে পাসওয়ার্ড হ্যাশ (Hash) করে ডাটাবেসে স্টোর করি। লগইনের সময় ইউজারের দেওয়া পাসওয়ার্ডকে আবার হ্যাশ করে ডাটাবেসের হ্যাশড পাসওয়ার্ডের সাথে কম্পেয়ার করা হয়।

### Q242: রেসপন্সিভ ডিজাইনের জন্য কি পদ্ধতি ফলো করেছেন?
Answer:
আমি Tailwind CSS এর 'Mobile-first' অ্যাপ্রোচ ফলো করেছি। আমি ব্রেকপয়েন্টস (sm, md, lg, xl) ব্যবহার করে প্রতিটি কম্পোনেন্টকে আলাদা আলাদা স্ক্রিন সাইজের জন্য ডিজাইন করেছি যাতে এটি মোবাইল এবং ডেস্কটপ উভয় জায়গাতেই সুন্দর দেখায়।

### Q243: আপনি কি আপনার প্রজেক্টে কোনো গ্লোবাল এরর হ্যান্ডলিং করেছেন?
Answer:
হ্যাঁ, আমি নেক্সট-জেএস এর `error.js` এবং `not-found.js` স্পেশাল ফাইলগুলো ব্যবহার করেছি। এর ফলে অ্যাপের যেকোনো জায়গায় অপ্রত্যাশিত এরর আসলে ইউজার একটি সুন্দর এরর পেজ দেখতে পায় এবং সেখান থেকে আবার ট্রাই করতে পারে।

### Q244: টেম্পলেট মার্কেটপ্লেসে পেমেন্ট হওয়ার পর ইউজার কীভাবে টেম্পলেটটি ডাউনলোড বা ইমপোর্ট করে?
Answer:
পেমেন্ট সাকসেস হলে আমি ডাটাবেসে একটি `TemplatePurchase` রেকর্ড তৈরি করি। ইউজার যখন ড্যাশবোর্ডে যায়, সে ওই টেম্পলেটের আইডিতে এক্সেস পায়। সেখানে ক্লিক করলে আমার বিল্ডার ওই টেম্পলেটের লেআউট ডাটা লোড করে এবং ইউজারের জন্য একটি নতুন প্রজেক্ট হিসেবে ক্লোন করে।

### Q245: প্রজেক্টে ইমেইল পাঠানোর জন্য কি ব্যবহার করেছেন?
Answer:
আমি Nodemailer ব্যবহার করেছি। পাসওয়ার্ড রিসেট লিংক এবং ইমেইল ভেরিফিকেশনের জন্য আমি জিমেইল এসএমটিপি (SMTP) কনফিগার করেছি। সুন্দর ইমেইল টেম্পলেটের জন্য আমি এইচটিএমএল এবং সিএসএস ইনলাইন স্টাইল ব্যবহার করেছি।

### Q246: নেক্সট-জেএস এর মিডলওয়্যারে কি কি কাজ করেছেন?
Answer:
মিডলওয়্যারে আমি প্রধানত দুটি কাজ করেছি: ১. অথেনটিকেশন চেক করে আনঅথরাইজড ইউজারদের লগইন পেজে পাঠিয়ে দেওয়া, এবং ২. নির্দিষ্ট কিছু এপিআই রাউটের ওপর রেট লিমিটিং অ্যাপ্লাই করা যাতে সিস্টেম স্প্যামিং থেকে মুক্ত থাকে।

### Q247: আপনি আপনার প্রজেক্টে কীভাবে আইকন ব্যবহার করেছেন?
Answer:
আমি Lucide React লাইব্রেরি ব্যবহার করেছি। এটি অনেক হালকা এবং প্রতিটি আইকন রিয়্যাক্ট কম্পোনেন্ট হিসেবে পাওয়া যায়, যা Tailwind দিয়ে খুব সহজেই স্টাইল করা যায়। এটি এসভিজি হওয়ার কারণে সব সাইজে শার্প দেখায়।

### Q248: প্রজেক্টের এনভায়রনমেন্ট ভেরিয়েবল কীভাবে ম্যানেজ করেন?

লোকালি আমি `.env.local` ফাইল ব্যবহার করি। প্রোডাকশনে Vercel এর ড্যাশবোর্ড থেকে ভেরিয়েবলগুলো সেট করে দিয়েছি। আমি সবসময় নিশ্চিত করি যাতে সিক্রেট কী-গুলো কখনোই গিটহাবে পুশ না হয়।

### Q249: আপনার প্রজেক্টটি কি স্কেলেবল? ভবিষ্যতে আরও ইউজার আসলে কীভাবে সামলাবেন?

### Q250: একজন ডেভেলপার হিসেবে এই প্রজেক্ট থেকে আপনার সবচেয়ে বড় শিক্ষা কি?

সবচেয়ে বড় শিক্ষা ছিল একটি বড় এবং রিয়েল-ওয়ার্ল্ড প্রজেক্ট কীভাবে শুরু থেকে শেষ পর্যন্ত প্ল্যানিং এবং ইমপ্লিমেন্ট করতে হয়। আমি শিখেছি কীভাবে ফ্রন্টএন্ড এবং ব্যাকএন্ডের মধ্যে ডাটা সিঙ্ক্রোনাইজ করতে হয় এবং ইউজার এক্সপেরিয়েন্স ঠিক রেখে পারফরম্যান্স অপ্টিমাইজ করা কতটা জরুরি।
